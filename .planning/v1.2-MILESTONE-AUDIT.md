## Integration Defects

### DEFECT-01: Typecheck CI Job Broken (Critical)

**Affected:** CIPL-01, CIPL-03

Phase 21 integrity tests (`parse-schema.ts`, `table-sync.test.ts`) use ES2018+ regex `s` flag and Set spread syntax. `tsconfig.json` has no `target` field (defaults to ES3). Running `pnpm typecheck` (`tsc --noEmit`) exits 1 with 4 errors:

```
parse-schema.ts(30,72): error TS1501: regex flag only available when targeting 'es2018' or later
table-sync.test.ts(26,48): error TS1501: regex flag only available when targeting 'es2018' or later
table-sync.test.ts(162,26): error TS2802: Set iteration requires '--target' of 'es2015' or higher
table-sync.test.ts(170,23): error TS2802: Set iteration requires '--target' of 'es2015' or higher
```

**Impact:** The `typecheck` CI job runs unconditionally. ci-pass gate checks it for `success`. Every PR to main will fail CI.

**Fix:** Add `"target": "ES2020"` to `tsconfig.json` `compilerOptions`.

### DEFECT-02: Benchmark Baseline Filepath Mismatch (Warning)

**Affected:** BNCH-01

`benchmarks/results.json` contains absolute worktree paths (`/home/ryan/.../worktrees/agent-a4dfb4bb/...`). In CI, paths differ. If vitest `--compare` matches by filepath-derived ID, baseline comparison will silently produce no output.

**Fix:** Regenerate `benchmarks/results.json` from non-worktree environment.

### DEFECT-03: Incomplete Drift Check (Minor)

**Affected:** SCHN-04

Supply chain drift check loop in ci.yml verifies `minimumReleaseAge`, `trustPolicy`, `blockExoticSubdeps` but omits `auditLevel`.

**Fix:** Add `auditLevel` to the `for setting in` loop.
