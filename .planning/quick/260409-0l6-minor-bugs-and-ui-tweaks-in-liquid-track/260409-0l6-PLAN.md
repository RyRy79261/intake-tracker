  <files>src/components/liquids/preset-tab.tsx</files>
  <action>
The bug is in `buildComposableEntry()` (line ~220). It creates BOTH:
1. An explicit water intake in `intakes[]` from `waterContentPercent` (e.g., 186ml for 190ml at 98%)
2. A `substance` with `volumeMl` set â€” which triggers `addComposableEntry` to auto-create ANOTHER water intake (line 125 of composable-entry-service.ts)

Fix: In `buildComposableEntry`, when building the substance object, do NOT include `volumeMl` if explicit water intakes are already being created (i.e., when `waterAmount > 0`). The substance record should still be created for caffeine/alcohol tracking â€” just without `volumeMl` so it does not trigger the auto-water-intake path.

Specifically, change the substance construction around lines 262-267 and 272-277:
- For caffeine substances: only include `volumeMl` when `waterAmount <= 0`
- For alcohol substances: only include `volumeMl` when `waterAmount <= 0`

This ensures the explicit `intakes[]` water record (which correctly applies `waterContentPercent`) is the ONLY water record created. The substance record still tracks caffeine/alcohol amounts correctly without `volumeMl`.
