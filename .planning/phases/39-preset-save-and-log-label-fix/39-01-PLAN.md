### Task 1: Make `addLiquidPreset` return the generated UUID

<task id="39-01-01">

<read_first>
- src/stores/settings-store.ts (full file - see current addLiquidPreset at lines 263-269, interface at line 113)
</read_first>

<action>
In `src/stores/settings-store.ts`, make two changes:

1. **Change the `SettingsActions` interface** (line 113):
   - FROM: `addLiquidPreset: (preset: Omit<LiquidPreset, "id">) => void;`
   - TO: `addLiquidPreset: (preset: Omit<LiquidPreset, "id">) => string;`

2. **Change the implementation** (lines 263-269):
   - FROM:
     ```typescript
     addLiquidPreset: (preset) =>
       set((state) => ({
         liquidPresets: [
           ...state.liquidPresets,
           { ...preset, id: crypto.randomUUID() },
         ],
       })),
     ```
   - TO:
     ```typescript
     addLiquidPreset: (preset) => {
       const id = crypto.randomUUID();
       set((state) => ({
         liquidPresets: [
           ...state.liquidPresets,
           { ...preset, id },
         ],
       }));
       return id;
     },
     ```

This changes the arrow function from an implicit return of `void` (from `set()`) to a block body that generates the ID first, calls `set()`, then returns the ID string.
</action>

**Why `setSelectedPresetId(newPresetId)`:** The `buildComposableEntry` function reads `selectedPresetId` state to construct `source: \`preset:${selectedPresetId ?? "manual"}\``. However, React state updates via `setState` are asynchronous -- they don't take effect until the next render. We need the value available synchronously.

**Actually, React state won't be available synchronously.** So instead, we need to pass the ID directly. The better approach is:

   - TO (corrected):
     ```typescript
     const newPresetId = addPreset({
       name: beverageName.trim(),
       tab,
       defaultVolumeMl: volumeMl,
       waterContentPercent,
       ...(caffeinePer100ml > 0 && { caffeinePer100ml }),
       ...(alcoholPer100ml > 0 && { alcoholPer100ml }),
       ...(saltPer100ml > 0 && { saltPer100ml }),
       isDefault: false,
       source: aiLookupUsed ? "ai" : "manual",
     });
     // Override selectedPresetId for this call since setState is async
     const prevSelectedPresetId = selectedPresetId;
     // Temporarily set for synchronous read by buildComposableEntry - not needed,
     // instead pass directly:
     ```

**The correct fix** requires modifying `buildComposableEntry` to accept an optional override parameter. Change the function signature and usage:

**Step A:** Modify `buildComposableEntry` (lines 205-273) to accept an optional `presetIdOverride` parameter:

   - FROM: `const buildComposableEntry = (): ComposableEntryInput => {`
   - TO: `const buildComposableEntry = (presetIdOverride?: string): ComposableEntryInput => {`

   And change the two places where `selectedPresetId` is used for the source field:

   - Line 213 FROM: `groupSource: \`preset:${selectedPresetId ?? "manual"}\`,`
   - Line 213 TO: `groupSource: \`preset:${presetIdOverride ?? selectedPresetId ?? "manual"}\`,`

   - Line 224 FROM: `source: \`preset:${selectedPresetId ?? "manual"}\`,`
   - Line 224 TO: `source: \`preset:${presetIdOverride ?? selectedPresetId ?? "manual"}\`,`

**Step B:** Update `handleSaveAndLog` to pass the new preset ID:

   - FROM: `const entry = buildComposableEntry();`
   - TO: `const entry = buildComposableEntry(newPresetId);`

**Step C:** Existing callers (`handleLog` at line 279) do NOT need changes -- they call `buildComposableEntry()` without arguments, which falls back to `selectedPresetId ?? "manual"` as before.
</action>
