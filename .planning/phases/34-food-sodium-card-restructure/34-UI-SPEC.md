## Component Specifications

### Card Header Layout

```
┌────────────────────────────────────────────────┐
│ [🍴] FOOD + SODIUM          {daily}mg / {limit}mg │
│                                    today (sodium) │
│                                   24h: {rolling}mg │
├────────────────────────────────────────────────┤
│ [═══════════════progress bar════════════════]   │
└────────────────────────────────────────────────┘
```

**Structure:**
- Left: icon (Utensils, `w-5 h-5`, in `bg-orange-100` rounded-lg pill) + "FOOD + SODIUM" title
- Right: sodium metric block (matches liquids card water metric pattern exactly)
  - Line 1: `{dailyTotal}mg / {limit}mg` — `text-sm font-medium`, red if over limit
  - Line 2: `today (sodium)` — `text-xs text-muted-foreground`
  - Line 3: `24h: {rollingTotal}mg` — `text-xs text-muted-foreground/70`
- Below header: sodium `Progress` bar, `h-3`, amber-to-orange gradient (salt theme), red when over limit

### AI Food Input

No visual changes. Existing input preserved exactly:
- `Input` component, `h-10 pr-10`, placeholder "Describe food for AI parsing..."
- Sparkles icon button inside input right side
- Loader2 spinner when parsing

### Composable Preview

No visual changes. Existing `ComposablePreview` component preserved as-is.

### Manual Details Collapsible

Existing structure preserved with one addition:

```
┌─ Add details (Collapsible) ────────────────────┐
│ What I ate (optional)           [textarea]      │
│ Weight in grams (optional)      [input]         │
│ Sodium source (optional)        [preset pills]  │
│ When                            [datetime]      │
│ [══════ Record with details ═══════]            │
└────────────────────────────────────────────────┘
```

**Sodium source preset pills** (new addition):
- Location: After "Weight in grams" input, before "When" datetime
- Label: "Sodium source (optional)" — `text-sm` Label component
- Layout: `flex flex-wrap gap-2` row of `Button variant="outline" size="sm"` pills
- Each pill shows preset name + sodium percent if < 100%: `{name}` or `{name} {percent}%`
- Active state: `bg-amber-100 border-amber-300 dark:bg-amber-900/50 dark:border-amber-700` (salt theme `activeToggle`)
- Inactive state: `opacity-70`
- Optional "None" default selection (no sodium source)
- When a preset is selected and grams are entered, show calculated sodium below:
  - `text-xs text-muted-foreground`: `"= {calculatedMg}mg sodium"`

### Merged History List

Single `RecentEntriesList` replacing the current two separate lists.

**Entry layout per row:**
```
┌────────────────────────────────────────────────┐
│ 2:30 PM   500mg Na   Chicken sandwich    [🗑]  │
│ 1:15 PM   250g       Pasta with sauce    [🗑]  │
│ 12:00 PM  800mg Na   2 minute noodles    [🗑]  │
└────────────────────────────────────────────────┘
```

**Columns (flex row with gap-2):**
1. **Time**: `text-sm text-muted-foreground shrink-0` — formatted via `formatDateTime`
2. **Amount** (when available): `text-xs font-medium`
   - Salt entries: `{amount}mg Na`
   - Eating entries with grams: `{grams}g`
   - Eating entries without grams: omitted
3. **Description/name**: `text-xs text-muted-foreground/70 truncate`
   - Salt entries: source note if available
   - Eating entries: `record.note` (the food description) — this IS the primary title per FOOD-02
   - Max width: `truncate` (CSS text-overflow: ellipsis)
4. **Delete button**: existing ghost button with Trash2 icon, `h-6 w-6`

**Row interaction:**
- Entire row clickable when `onEdit` provided
- Hover: `hover:bg-black/5 dark:hover:bg-white/5`
- Active: `active:bg-black/10 dark:active:bg-white/10`
- Click on eating entry: opens `EditEatingDialog`
- Click on salt entry: opens `EditIntakeDialog`
- Delete button: `e.stopPropagation()` to prevent edit trigger

### Quick Nav Item

After restructure, the quick nav shows a single "Eating" item that scrolls to the unified card:
- Icon: Utensils
- Label: "Eating"
- Scrolls to: `section-food-salt` (via updated `eating.sectionId`)
- The "Sodium" (Sparkles) quick nav item remains in the footer but scrolls to `section-salt` which has no matching DOM element (silently no-ops, same as current behavior)

## Interaction Specifications

### Sodium Progress Bar
- Animates on value change (existing Progress component behavior)
- Color transitions: amber-to-orange gradient under limit, solid red-500 when over limit

### Manual Details + Sodium Presets
- Selecting a sodium preset is optional
- Preset selection persists within the collapsible session (resets when collapsed/reopened)
- Calculated sodium shown inline only when both preset and grams are provided
- Submit creates a composable entry: eating record + optional sodium intake record

### History Entry Edit/Delete
- Tap row: opens appropriate edit dialog based on entry type
- Tap trash icon: soft-delete with 5-second undo toast
- No visual distinction between eating and sodium entries (unified appearance)

### AI Parse Flow
- No changes to existing AI parse → preview → confirm flow
- ComposablePreview continues to handle multi-record preview
