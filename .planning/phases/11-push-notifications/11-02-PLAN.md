    1. Create `src/app/api/push/send/route.ts`:
       - This is the cron-triggered endpoint. Auth via `CRON_SECRET` header, NOT withAuth (no user session).
       - Check `Authorization: Bearer <CRON_SECRET>` header. If mismatch, return 401.
       - Also allow LOCAL_AGENT_MODE bypass for testing (same pattern as other routes but simpler).
       - Get current time as HH:MM string and today as YYYY-MM-DD string. Get dayOfWeek (0-6).
       - **Initial notifications**: Call `getDueNotifications(currentTime, dayOfWeek, today)`. For each result, call `sendPush` with payload `JSON.stringify({ title: "Time for your {time_slot} medications", body: row.medications_json, tag: "dose-{time_slot}", url: "/medications?tab=schedule" })`. On success, call `logSentNotification(row.user_id, row.time_slot, today, 0)`. On 410, call `deletePushSubscription(row.user_id)`.
       - **Follow-up reminders**: For each unique user_id from active subscriptions, call `getSettings(userId)` to get followUpCount and followUpIntervalMinutes. Loop from 1 to followUpCount: call `getFollowUpNotifications(today, i, settings.followUpIntervalMinutes)`. For each result, send push with same payload format but title "Reminder: your {time_slot} medications". On success, logSentNotification with follow_up_index=i. On 410, delete subscription.
       - Return 200 `{ sent: totalSent, followUps: totalFollowUps }`

    2. Extend `src/lib/push-notification-service.ts`:
       - Add at the top (below existing imports): `urlBase64ToUint8Array(base64String: string): Uint8Array` utility function per research doc (padding + replace + atob + Uint8Array.from)
       - Add `export async function subscribeToPush(authToken: string): Promise<PushSubscription | null>`:
         - Guard: if `!("serviceWorker" in navigator) || !("PushManager" in window)` return null
         - `const registration = await navigator.serviceWorker.ready`
         - Check existing: `let subscription = await registration.pushManager.getSubscription()`
         - If exists, still send to server (in case server lost it): POST to `/api/push/subscribe` with subscription.toJSON() body and Bearer authToken header
         - If not exists, create: `subscription = await registration.pushManager.subscribe({ userVisibleOnly: true, applicationServerKey: urlBase64ToUint8Array(process.env.NEXT_PUBLIC_VAPID_PUBLIC_KEY!) })`
         - POST to `/api/push/subscribe` with `{ endpoint: subscription.endpoint, keys: { p256dh: subscription.toJSON().keys?.p256dh, auth: subscription.toJSON().keys?.auth } }` and Bearer authToken
         - Return subscription
       - Add `export async function unsubscribeFromPush(authToken: string): Promise<boolean>`:
         - Get registration, get existing subscription
         - If subscription exists, call `subscription.unsubscribe()`
         - POST to `/api/push/unsubscribe` with Bearer authToken
         - Return true on success, false on error
       - Keep ALL existing exports unchanged (isNotificationSupported, getNotificationPermission, requestNotificationPermission, showNotification, etc.)
  </action>
