export function useWeightRecords(limit: number = 5) {

To:
```typescript
export function useWeightRecords(limit: number = 5) {
  return useLiveQuery(() => getWeightRecords(limit), [limit]);
}
```

By removing the third argument (`[]`), `useLiveQuery` returns `undefined` before the Dexie query resolves. This allows consumers to distinguish between:
- `undefined` — query has not yet resolved (loading)
- `[]` — query resolved, zero records exist
- `WeightRecord[]` — query resolved with data

The return type changes from `WeightRecord[]` to `WeightRecord[] | undefined`. This is compatible with `RecentEntriesList` which already accepts `T[] | undefined` for its `records` prop.
