No code changes are needed — this plan confirms OBS-01 is already satisfied by existing infrastructure from prior phases.
</objective>

<action>
This is a validation-only task. No files are created or modified.

Verify the following chain is intact by reading each file:

1. **package.json** contains `"version": "X.Y.Z"` (currently `"1.2.0"` — managed by Release Please from Phase 27)

2. **next.config.js** line 59 contains:
   ```js
   NEXT_PUBLIC_APP_VERSION: packageJson.version,
   ```
   This injects the package.json version as a build-time env var.

3. **src/components/about-dialog.tsx** line 16 contains:
   ```ts
   const appVersion = process.env.NEXT_PUBLIC_APP_VERSION || "0.0.0";
   ```
   And the JSX renders `{appVersion}` in a "Version" row.

4. **src/app/settings/page.tsx** line 80 renders `<AboutDialog />` inside the settings layout.

5. **src/app/api/version/route.ts** returns:
   ```ts
   { version: process.env.NEXT_PUBLIC_APP_VERSION || "0.0.0", gitSha: ..., environment: ... }
   ```

6. **src/hooks/use-version-check.ts** fetches from `/api/version` every 5 minutes and compares `serverVersion` against `CLIENT_VERSION` (which reads `NEXT_PUBLIC_APP_VERSION`).

If all 6 links are present, OBS-01 is satisfied. Report the validation results.
</action>
