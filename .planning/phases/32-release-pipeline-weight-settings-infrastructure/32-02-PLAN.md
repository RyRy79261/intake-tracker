Add a troubleshooting comment block at the top of `.github/workflows/release-please.yml`, before the `name:` line. The full file should become:

```yaml
# Troubleshooting: If release-please fails to create or update PRs:
#
# 1. Go to GitHub repo Settings > Actions > General > Workflow permissions
# 2. Select "Read and write permissions"
# 3. Check "Allow GitHub Actions to create and approve pull requests"
# 4. Save
#
# The `permissions` block below is necessary but NOT sufficient — the repo-level
# setting must also grant write access. YAML permissions cannot exceed repo-level grants.

name: Release Please

on:
  push:
    branches: [main]

permissions:
  contents: write
  pull-requests: write

jobs:
  release-please:
    runs-on: ubuntu-latest
    steps:
      - uses: googleapis/release-please-action@v4
        with:
          release-type: node
```

This preserves the existing workflow exactly while adding discoverable troubleshooting notes.

  <action>
**Manual step — user must perform this in GitHub UI:**

1. Go to the GitHub repository: https://github.com/RyRy79261/intake-tracker/settings/actions
2. Scroll to "Workflow permissions" section
3. Select **"Read and write permissions"** (not "Read repository contents and packages permissions")
4. Check **"Allow GitHub Actions to create and approve pull requests"**
5. Click **Save**

**Verification after saving:**
- Push a conventional commit (e.g., `fix: test release-please`) to main
- Check the Actions tab for the "Release Please" workflow run
- Verify it either creates a new release PR or updates an existing one without permission errors

If the release-please action has never run successfully, it will create a new PR on the next conventional commit push. If it previously created a PR that is still open, it will update that PR.

  </acceptance_criteria>
  <done>GitHub Actions workflow permissions configured to allow release-please to create PRs.</done>
