   - Import `withAuth` from `@/lib/auth-middleware`

   - Remove ALL inline auth logic (isPrivyConfigured checks, authToken extraction, verifyAndCheckWhitelist calls, clientApiKey fallback)

3. Migrate `src/app/api/ai/substance-enrich/route.ts`:
   - Same pattern: wrap with `withAuth`, remove inline auth
   - Use centralized `sanitizeForAI()` for input sanitization

4. Update `src/app/api/ai/status/route.ts`:
   - Keep as public GET endpoint (no auth required -- per research, it exposes only boolean config flags)
   - Remove `serverApiKeyLength` from the response (leaks info about key size)
   - Keep `privyConfigured`, `serverApiKeyConfigured`, `env` booleans

   - Add South African ID number pattern: 13 consecutive digits
   - Keep existing patterns (email, US phone, SSN)

6. Ensure all 3 POST routes consistently call `sanitizeForAI()` on user-provided text input before sending to Perplexity.
