Create a new file `src/components/medications/compound-details-drawer.tsx`.

This drawer displays compound-level data from the Prescription record in read-only mode, with an AI refresh button that fetches updated data and presents a field-by-field diff.

**Component structure:**

```typescript
"use client";

import { useState } from "react";
import { Drawer, DrawerContent, DrawerHeader, DrawerTitle } from "@/components/ui/drawer";
import { Button } from "@/components/ui/button";
import { Checkbox } from "@/components/ui/checkbox";
import { Loader2, RefreshCw } from "lucide-react";
import { useMedicineSearch } from "@/hooks/use-medicine-search";
import { useUpdatePrescription } from "@/hooks/use-medication-queries";
import { useToast } from "@/hooks/use-toast";
import type { Prescription } from "@/lib/db";

interface CompoundDetailsDrawerProps {
  prescription: Prescription | null;
  open: boolean;
  onOpenChange: (open: boolean) => void;
}

interface FieldDiff {
  field: string;
  label: string;
  oldValue: string | string[] | undefined;
  newValue: string | string[] | undefined;
  accepted: boolean;
}
```

**Sections (read-only, per UI-SPEC):**

Display these compound fields from the prescription, each as a labeled section:
- Drug Class (`prescription.drugClass`)
- Mechanism of Action (`prescription.mechanismOfAction`)
- Common Indications (`prescription.commonIndications` -- array, render as comma-separated or bulleted list)
- Dosage Strengths (`prescription.dosageStrengths` -- array)
- Food Instructions (`prescription.foodInstruction` + `prescription.foodNote`)
- Contraindications (`prescription.contraindications` -- array)
- Warnings (`prescription.warnings` -- array)

Section heading style per UI-SPEC: `text-xs font-semibold text-muted-foreground uppercase tracking-wide`. Section content: `text-sm font-medium`.

**Empty state:** If no compound data exists (all fields undefined), show:
- Heading: "No compound data"
- Body: "Tap 'Refresh from AI' to fetch drug information."

**AI Refresh flow:**

1. "Refresh from AI" button at bottom (full-width, secondary variant per UI-SPEC)
2. On click: call `searchMutation.mutateAsync({ query: prescription.genericName })` using the `useMedicineSearch` hook
3. Loading state: button shows spinner + "Fetching..."
4. On success: compare returned fields with existing prescription fields. Build a `FieldDiff[]` array for fields that differ.
5. If no differences: show toast "Compound data is up to date."
6. If differences: show diff UI:
   - Each changed field: old value (line-through, text-muted-foreground) vs new value (text-foreground, bg-green-50 dark:bg-green-950 highlight)
   - Checkbox per field to accept/reject
   - "Apply Changes" primary button (applies only checked fields via `updatePrescription.mutateAsync`)
   - "Discard Changes" ghost button (dismisses all proposed changes, preserves existing data per UI-SPEC)
7. On error: toast "Could not fetch compound data. Check your connection and try again."

**Field mapping from AI response to Prescription fields:**
- `drugClass` -> `drugClass`
- `mechanismOfAction` -> `mechanismOfAction`
- `commonIndications` -> `commonIndications`
- `dosageStrengths` -> `dosageStrengths`
- `foodInstruction` -> `foodInstruction`
- `foodNote` -> `foodNote`
- `contraindications` -> `contraindications`
- `warnings` -> `warnings`

**Diff comparison:**
For string fields: compare with `===`.
For array fields: compare with `JSON.stringify(sorted)`.

The drawer should follow the existing pattern: `max-h-[90dvh] flex flex-col` with scrollable content area.
