## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px (p-2, gap-2) | Compact element spacing, badge padding |

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Caption | 12px | medium (500) | 1.33 | text-xs font-medium |
| Body | 14px | medium (500) | 1.43 | text-sm font-medium |
| Label | 14px | semibold (600) | 1.43 | text-sm font-semibold |
| Heading | 18px | semibold (600) | 1.33 | text-lg font-semibold |

Notes:
- This project is mobile-first (max-w-lg container). The dominant text sizes are text-xs (305 uses) and text-sm (240 uses), reflecting a data-dense health tracking UI.
- font-medium (500) is the project's "normal" weight (132 uses). font-semibold (600) is the emphasis weight (81 uses). font-bold (700) exists but is rare (12 uses) -- avoid introducing new font-bold unless matching an existing card header pattern.
- Do NOT introduce text-base (16px) as a primary size -- it is only used 3 times in the entire codebase.

## Color

All colors are HSL CSS variables defined in globals.css with light and dark mode variants.

| Role | Token | Light Value | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | --background | hsl(220 20% 97%) | Page background, surfaces |
| Secondary (30%) | --card / --secondary | hsl(0 0% 100%) / hsl(220 15% 92%) | Cards, muted areas, nav |
| Accent (10%) | --primary | hsl(220 70% 50%) | Primary CTA buttons, focus rings, active tab indicators |
| Destructive | --destructive | hsl(0 84% 60%) | Delete buttons, over-limit warnings, destructive confirmations |

### Domain-Specific Tokens (existing, used in this phase)

| Token | HSL (light) | Usage in Phase 1 |
|-------|-------------|-------------------|
| --caffeine | hsl(48 96% 53%) | Coffee preset accordion header color (D-22) |
| --alcohol | hsl(292 84% 61%) | Alcohol preset accordion header color (D-22) |
| --medication | hsl(168 76% 36%) | Medication schedule items, dose rows |

### New Color: Mixed Substance (D-22)

| Token | Value | Usage |
|-------|-------|-------|
| orange-500 | Tailwind orange-500 (no CSS variable needed) | Mixed caffeine+alcohol preset items only |

Note: D-22 specifies "orange for mixed" items. Since this is a single-use color for a small number of mixed substance presets, use Tailwind's built-in orange-500 directly rather than creating a new CSS variable. Apply as `text-orange-500` on the accordion trigger text/icon for mixed items.

**Accent reserved for:**
- Primary CTA buttons (Save, Confirm, Add)
- Active tab indicators in medication navigation
- Focus ring on form inputs
- Progress bar fills (domain-specific tokens override for water/salt/medication)

## Visual Hierarchy

This phase modifies existing screens rather than creating new ones. The most heavily changed screen is the Settings page (D-19 through D-23), which gains a new accordion-based layout replacing the modal customization panel.

### Settings Page — Primary Focal Point

| Property | Value |
|----------|-------|
| Primary anchor | The accordion section group (Water, Coffee, Alcohol, Food/Beverage) |
| Placement | Below existing behavior settings (limits, increments, day-start-hour, theme) |
| Visual weight | Domain-colored accordion triggers (--caffeine, --alcohol, orange-500, foreground) draw the eye to the preset management area as the primary interactive zone |
| Supporting context | Existing behavior settings above the accordion use muted card backgrounds (--card), establishing visual subordination so the accordion stands out as the new content area |

**Rationale:** The accordion is the only net-new structural UI element on any screen in this phase. Existing behavior settings retain their current styling; the accordion's domain-colored triggers create a natural focal point without modifying the surrounding layout.

## Copywriting Contract

### Phase-Wide Patterns

This phase is bug fixes and UX improvements -- it modifies existing UI rather than creating new flows. Copywriting contracts are per-decision.

| Element | Copy | Decision |
|---------|------|----------|
| Dose format | "{fraction} tablet ({computed_dose}{unit})" e.g. "1/2 tablet (6.25mg)" | D-07, D-10 |
| Collapsed Rx summary | "{pillAmount} {numberOfTimes} per day" e.g. "1/2 tablet 2x per day" | D-09 |
| Brand display in schedule | "{compoundName} ({brandName})" e.g. "Carvedilol (Coreg)" | D-08 |
| Edit button rename | "Compound Details" (was "Edit") | D-12 |
| AI refresh button | "Refresh from AI" | D-12 |
| Progress bar format | "{taken}/{total} Taken" e.g. "3/6 Taken" | D-06 |

### Accordion Headers (D-20, D-22)

| Section | Header Copy | Icon |
|---------|-------------|------|
| Water presets | "Water Presets" | Droplets (lucide) |
| Coffee presets | "Coffee Presets" | Coffee (lucide) |
| Alcohol presets | "Alcohol Presets" | Wine (lucide) |
| Food/Beverage presets | "Food & Beverage Presets" | UtensilsCrossed (lucide) |

### Empty States

| Context | Heading | Body |
|---------|---------|------|
| Compound Details drawer (no AI data yet) | "No compound data" | "Tap 'Refresh from AI' to fetch drug information." |
| Accordion section (no presets) | "No presets" | "Add presets from the main dashboard cards." |
| Insight thresholds (first use) | -- | Thresholds pre-populated with defaults; no empty state needed |

### Error States

| Context | Copy |
|---------|------|
| AI refresh failure | "Could not fetch compound data. Check your connection and try again." |
| AI refresh partial | "Some fields could not be updated. Review the changes below." |

### Destructive Actions

| Action | Trigger | Confirmation |
|--------|---------|--------------|
| Delete preset (D-21, D-23) | Red X icon button on preset row | No confirmation dialog -- immediate delete with toast "Preset deleted" + undo action (3s timeout) |

## New Component Interactions

### Accordion: Settings Restructure (D-20)

| Property | Value |
|----------|-------|
| Component | shadcn/ui Accordion (install: `npx shadcn@latest add accordion`) |
| Type | `type="multiple"` (multi-open) |
| Default state | All collapsed |
| Animation | Default Radix accordion animation (smooth height transition) |
| Trigger styling | Domain-colored text: `text-caffeine` for coffee, `text-alcohol` for alcohol, `text-orange-500` for mixed, `text-foreground` for water/food |
| Content | Preset list with delete-only buttons (red X, no edit) |

### Compound Details Drawer (D-12)

| Property | Value |
|----------|-------|
| Component | Existing shadcn/ui Drawer pattern |
| Max height | 90dvh (existing drawer pattern) |
| Mode | Read-only by default |
| Layout | Sections in scroll area: Drug Class, Mechanism of Action, Common Indications, Dosage Strengths, Food Instructions |
| Section headings | text-xs font-semibold text-muted-foreground uppercase tracking-wide |
| Section content | text-sm font-medium |
| Indication text (D-11) | Truncated to 2 lines with "Show more" / "Show less" toggle |
| AI Refresh button | Bottom-fixed, full-width, secondary variant: "Refresh from AI" |

### AI Refresh Diff (D-12, Claude's Discretion)

| Property | Value |
|----------|-------|
| Trigger | "Refresh from AI" button in Compound Details drawer |
| Loading state | Button shows spinner + "Fetching..." |
| Diff display | Field-by-field comparison: old value (line-through, text-muted-foreground) vs new value (text-foreground, bg-green-50 dark:bg-green-950 highlight) |
| Actions per field | Checkbox to accept/reject each changed field |
| Footer | "Apply Changes" primary button (applies only checked fields) + "Discard Changes" ghost button (dismisses all proposed AI changes, preserves existing compound data) |
| No changes state | Toast: "Compound data is up to date." |

### Insight Thresholds (D-18)

| Property | Value |
|----------|-------|
| Trigger | Gear icon (Settings lucide icon, 16px) next to each insight type header |
| UI | Inline numeric input that appears below the insight header on gear click |
| Input | shadcn NumericInput with label e.g. "Alert below [80]%" |
| Persistence | Zustand settings store (version bump to 5) |
| Default thresholds | Adherence: 80%, Water: 2000ml, other insights: contextual |

### Medication Tab Wrap (D-02)

| Property | Value |
|----------|-------|
| Current | Single-row flex with overflow (tabs get cramped) |
| Fix | `flex-wrap` on the tab list container |
| Tab sizing | Remove any `flex-shrink-0` or `whitespace-nowrap` constraints that prevent wrapping |
| Visual | Second row aligns left, same gap as first row |

### Export Buttons Stack (D-17)

| Property | Value |
|----------|-------|
| Current | Side-by-side, each taking ~1/3 width |
| Fix | Stack vertically: `flex-col w-full` |
| Button width | Full width each: `w-full` |
| Gap | 8px (gap-2) between buttons |
