Modify `src/components/medications/add-medication-wizard.tsx` to add a non-blocking conflict check interstitial when saving a new prescription:

1. Import `useInteractionCheck` and `InteractionResult` from `@/hooks/use-interaction-check`.
2. Import `AlertTriangle` from `lucide-react`.

3. Add state: `const [conflictCheckState, setConflictCheckState] = useState<"idle" | "checking" | "warning" | "unavailable">("idle")`.
4. Get interaction check hook: `const { check, data: conflictData, isLoading: isCheckingConflicts, error: conflictError, reset: resetConflicts } = useInteractionCheck()`.

5. Modify the `handleSave` function:
   - At the start of handleSave, BEFORE any save logic, if `conflictCheckState === "idle"` and this is a NEW prescription (selectedPrescriptionId === "new"):
     - Set `conflictCheckState` to "checking"
     - Get active prescriptions via the existing `prescriptions` variable from `usePrescriptions()` (already available in the wizard)
     - If there are active prescriptions, call `check({ mode: "conflict", newMedication: genericName || searchQuery, activePrescriptions: prescriptions.filter(p => p.isActive).map(p => ({ genericName: p.genericName })) })`
     - After the check resolves:
       - If error or timeout: set `conflictCheckState` to "unavailable"
       - If data has interactions with severity AVOID or CAUTION: set `conflictCheckState` to "warning"
       - If no concerning interactions: proceed with original save logic (call original save)
     - Return early (don't execute save yet)
   - If `conflictCheckState === "warning"` or `conflictCheckState === "unavailable"`: proceed with original save logic (user already acknowledged)
   - For existing prescription assignments (selectedPrescriptionId !== "new"), skip conflict check entirely

6. Add a conflict warning overlay inside the drawer, rendered when `conflictCheckState === "warning"`:
   - A div overlaying the wizard content: `<div className="absolute inset-0 bg-background/95 z-10 flex flex-col p-4 overflow-y-auto">`
   - Header: `<div className="flex items-center gap-2 mb-4"><AlertTriangle className="w-5 h-5 text-amber-500" /><h3 className="text-sm font-semibold">Interaction Warning</h3></div>`
   - Summary text from `conflictData?.summary` if available
   - List each interaction from `conflictData?.interactions` filtered to AVOID and CAUTION:
     - AVOID items: `<div className="p-2 rounded-md bg-red-50 dark:bg-red-950/30 mb-2">` with red severity badge and description
     - CAUTION items: `<div className="p-2 rounded-md bg-amber-50 dark:bg-amber-950/30 mb-2">` with amber severity badge and description
   - Two buttons at bottom:
     - `<Button variant="outline" onClick={() => { resetConflicts(); setConflictCheckState("idle"); }}>Go Back</Button>`
     - `<Button className="bg-amber-600 hover:bg-amber-700" onClick={() => { setConflictCheckState("warning"); /* handleSave will proceed on next call */ handleSave(); }}>I'm Aware, Save Anyway</Button>`
   - Note: The "Save Anyway" button re-calls handleSave. Since conflictCheckState is "warning", the check at the top of handleSave will skip the conflict check and proceed to save.

7. Add a checking overlay when `conflictCheckState === "checking"`:
   - `<div className="absolute inset-0 bg-background/95 z-10 flex flex-col items-center justify-center p-4">`
   - `<Loader2 className="w-8 h-8 animate-spin text-muted-foreground mb-3" />`
   - `<p className="text-sm text-muted-foreground">Checking for interactions...</p>`

8. Add handling for `conflictCheckState === "unavailable"`:
   - Proceed to save immediately (AI check failed, don't block user)
   - This means the unavailable state auto-triggers save, no extra UI needed

9. Reset conflictCheckState to "idle" when the wizard opens or when the user navigates back to a previous step.
