Create `docs/architecture/` directory if it doesn't exist. Write `docs/architecture/neon-vercel.md` with the following sections:

**1. Overview** — Single paragraph: this project uses NeonDB via the Neon-Managed Vercel Integration. Only push notification infrastructure currently uses NeonDB (4 tables in `src/lib/push-db.ts`). All 16 health tracking tables remain client-side in IndexedDB via Dexie.js.

**2. Current State** — What NeonDB is used for today:
- Only consumer: `src/lib/push-db.ts` using `@neondatabase/serverless` (v1.0.2)
- Four tables: `push_subscriptions`, `push_dose_schedules`, `push_sent_log`, `push_settings`
- Access pattern: raw SQL via `neon()` driver, no ORM
- Server-only access via Next.js API routes (connection strings never reach client)
- Security: `src/__tests__/bundle-security.test.ts` verifies DATABASE_URL not leaked to client

**3. Branch Database Lifecycle** — Include a Mermaid diagram showing:
```
Neon Project
├── main (production) ─── Vercel Production environment
├── staging (child of main) ─── Vercel Staging environment
│   └── Reset from parent on release publish
└── preview/<git-branch> (child of main) ─── Vercel Preview deployment
    └── Auto-created on push, cleaned up on git branch deletion
```

Explain each branch type:
- **Production (main):** Default Neon branch. Connected to Vercel production environment. DATABASE_URL points here for production deploys. Snapshot created before staging-to-main promotion (via `promote-to-production.yml`).
- **Staging:** Persistent child of main. Reset to production state on release publish (via `staging-db-reset.yml` using `neondatabase/reset-branch-action@v1`). Reset is a complete overwrite of schema and data. Connections temporarily interrupted during reset.
- **Preview (preview/\<git-branch\>):** Auto-created via webhook when Vercel triggers preview deployment. Copy-on-write from main (instant, no data copying delay). Isolated connection string injected for that specific deployment only. Cleaned up when git branch is deleted (executes on next preview deploy creation).
