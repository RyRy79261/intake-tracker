## Key decisions this session

- **Sonnet + web_search everywhere** — user guardrail. Haiku is no longer acceptable for any user-facing AI route.
- **Parse route schema rename `salt` → `value_mg`** + explicit `measurement_type` — makes the sodium/salt contract unambiguous.
- **Set `sodiumSource` BEFORE `sodiumMg`** in food-section.tsx — dropdown must reflect the correct unit before the value is shown.
- **Sign-up redirect via `router.replace + refresh`** — `callbackURL` on `signUp.email` is server-flow metadata, not a client redirect.
- **`window.location` banned for in-app navigation** — sign-out converted to `router.replace/refresh`. `reload()` for SW/version updates and `.origin/.protocol` reads are still allowed.
- **`max_tokens` bump for web_search routes** — server tool blocks count toward output tokens. Each route got headroom; the bumps above are the per-route budget.

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Starting dev server automatically | User feedback: always let user run `pnpm dev` themselves | advisory | Never invoke `pnpm dev` in tool calls |
| Using Haiku for user-facing AI routes | World-knowledge gaps caused Mio Mio Mate 1.9 mg/100ml vs 20 actual | blocking | All user-facing AI routes must use `CLAUDE_MODELS.quality` and include `WEB_SEARCH_TOOL` |
| `window.location.href = ...` for in-app nav | Breaks Next.js routing, causes full page reload and state loss | blocking | Use `router.replace/refresh` from `next/navigation`; `window.location.reload()` is still OK for SW updates |
| Conflating sodium (Na) and salt (NaCl) in AI responses | Salt ≈ 2.5× sodium by mass; silent conversion causes wrong numbers | blocking | AI schema returns `value_mg` + explicit `measurement_type`; client does the conversion |
