## Key decisions made this session

- **Collapsible over Accordion** for expandable sub-sections — lets the caffeine/alcohol Switch sit as a sibling of the trigger (nested buttons would be invalid HTML). The `headerRight` slot on `ExpandableSettingsSection` is the extension point for this.
- **Don't strip existing section headers** — user correction: the existing visual header IS the trigger; don't replace with a generic wrapper header.
- **`substanceConfig` removal strategy**: Zustand persist silently drops unknown keys, so removing the field from the interface + a `delete state.substanceConfig` migration at version 6 cleanly sweeps existing localStorage. User confirmed their phone (main branch) only has localStorage, so nothing to worry about.
- **Account card at top of settings** — not nested in any accordion. Heading removed. Loader while `useSession` is pending.
