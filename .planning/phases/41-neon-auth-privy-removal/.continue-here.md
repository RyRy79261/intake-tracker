## Key decisions this session

- **Sonnet everywhere + web_search everywhere** — user's new guardrail: "everything we do needs a web tool and everything must use Sonnet as a minimum". Haiku is no longer acceptable for any user-facing AI route; its world-knowledge gaps produced wrong estimates on niche beverages.
- **Parse route field rename `salt` → `value_mg`** (bigger blast radius path chosen) — the previous field name conflated the two units and caused the model to confidently return `measurement_type: "sodium"` while its underlying number was salt mass. Rename makes the API contract unambiguous.
- **Set `sodiumSource` BEFORE `sodiumMg`** in food-section.tsx — React state updates are queued, but the dropdown must reflect the correct unit before the user sees the number next to it.
- **Sign-up redirect via `router.replace + refresh`** — callbackURL on `signUp.email` is server-flow metadata, not a client redirect. Explicit navigation is required. Same applies to `signIn.email` (but Scenario 1 passed, so sign-in's current behavior — where Better Auth client does its own navigation — is working).
- **`window.location` is banned for in-app navigation** — converted all sign-out callers to `router.replace/refresh`. Legitimate uses (`reload()` for SW/version updates, `.origin/.protocol` reads) are still allowed per memory inventory.
