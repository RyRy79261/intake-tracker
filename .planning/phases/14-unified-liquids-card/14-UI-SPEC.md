| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px | Compact element spacing, gap between preset grid buttons |

## Color

This phase uses **domain-specific color tokens** already defined in the project, not generic accent. The Liquids card dynamically themes per active tab.

### Tab-Specific Theming

| Tab | Theme Source | Gradient | Border | Button | Icon Color |
|-----|-------------|----------|--------|--------|------------|
| Water | `CARD_THEMES.water` | `from-sky-50 to-cyan-50` | `border-sky-200` | `bg-sky-600 hover:bg-sky-700` | `text-sky-600` |
| Beverage | `CARD_THEMES.water` | `from-sky-50 to-cyan-50` | `border-sky-200` | `bg-sky-600 hover:bg-sky-700` | `text-sky-600` |
| Coffee | `CARD_THEMES.caffeine` | `from-yellow-50 to-amber-50` | `border-yellow-200` | `bg-yellow-700 hover:bg-yellow-800` | `text-yellow-700` |
| Alcohol | `CARD_THEMES.alcohol` | `from-fuchsia-50 to-pink-50` | `border-fuchsia-200` | `bg-fuchsia-600 hover:bg-fuchsia-700` | `text-fuchsia-600` |

### Card Surface Color Contract

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` / active tab gradient | Card background gradient shifts per tab |
| Secondary (30%) | `bg-muted/50` | TabsList background, field containers, preset grid button resting state |
| Accent (10%) | Per-tab theme color | Active tab trigger, confirm button, progress bar fill, selected preset border |
| Destructive | `hsl(var(--destructive))` | Over-limit warning text on water tab, error toast |

### Accent Reserved For (per tab)

- Active tab trigger indicator (`data-[state=active]:bg-background data-[state=active]:shadow-sm`)
- Selected preset button border highlight (e.g. `bg-yellow-100 border-yellow-300` for coffee)
- Confirm/Log button fill color
- Progress bar indicator (water tab only)
- AI lookup icon button tint when idle
- Over-limit warning uses `text-red-600 dark:text-red-400` (not theme accent)

### Card Gradient Transition

When switching tabs, the card background gradient transitions smoothly. Use `transition-all duration-300` on the gradient wrapper (existing pattern from IntakeCard).

## Layout Specifications

### Card Header (all tabs)

```
[icon-box 36px] [8px gap] ["LIQUIDS" heading]          [daily total / limit]
                                                         [today label]
                                                         [24h: rolling total]
```

- Icon box: 36px (p-2 + w-5 h-5 icon = existing pattern)
- Header label: "LIQUIDS" — `font-semibold text-lg uppercase tracking-wide`
- Right side: daily total + limit for water tab; substance total for coffee/alcohol tabs; nothing for beverage tab
- Icon changes per active tab: `Droplets` (Water/Beverage), `Coffee` (Coffee), `Wine` (Alcohol)

### Tab Strip

```
[ Water | Beverage | Coffee | Alcohol ]
```

- Full-width `TabsList` with 4 `TabsTrigger` elements
- `TabsList` className: `w-full grid grid-cols-4` (equal width columns)
- Each trigger: `text-xs font-medium` (12px) to fit 4 tabs
- Active trigger: standard shadcn active style (`bg-background text-foreground shadow-sm`)
- Tab strip sits below the header, above tab content
- Margin: `mb-4` below header to tab strip, `mt-4` from tab strip to content (via TabsContent default mt-2 + additional spacing)

### Water Tab Layout

Exact lift of current IntakeCard water UX. Layout preserved:

```
[Progress bar h-3]                                    mb-4

[-] [  +250ml  ] [+]                                  gap-3
     "tap to edit"

[====== Confirm Entry (h-12) ======]                  mt-4

Recent entries list                                   mt-4
  10:30 AM    250ml
  09:15 AM    250ml    [coffee badge]
```

- Minus/Plus: `Button variant="outline" size="icon-lg"` (h-14, w-14, rounded-full)
- Center value: `text-3xl font-bold tabular-nums` on tappable surface
- Confirm: `Button` full-width, h-12, themed `bg-sky-600`
- Progress bar: `Progress` component, `h-3`, gradient fill
- Over-limit: progress turns `bg-red-500`, total turns `text-red-600`
- Would-exceed: pending amount turns `text-orange-600`

### Beverage Tab Layout

```
[Drink name input]                                    mb-3

[-] [  +250ml  ] [+]                                  gap-3
     "tap to edit"

[====== Log Beverage (h-12) ======]                   mt-4
```

- Name input: `Input` component, `h-10`, placeholder "e.g. Juice, Smoothie"
- +/- and center value: identical to water tab pattern
- Confirm button: "Log Beverage" label, themed `bg-sky-600`
- No progress bar, no limit display
- No recent entries (entries appear in water tab since they are water intake records)

### Coffee/Alcohol Tab Layout (PresetTab)

```
[Preset] [Preset]                                     2-col grid, gap-2
[Preset] [Preset]
[Preset] [Preset]                                     mb-3

[AI text input_____________________ (sparkle)]        mb-3

Volume (ml)          per 100ml
[___250___]          [___210___] mg/100ml     mb-2

Calculated: 525 mg caffeine                           mb-4

[== Log ==] [== Save & Log ==]                        gap-3
```

#### Preset Grid

- 2-column grid: `grid grid-cols-2 gap-2`
- Each preset button: `Button variant="outline" size="sm"` with min-height 40px
- Content: preset name (left-aligned, `text-sm font-medium`) + volume hint (right-aligned, `text-xs text-muted-foreground`)
- Layout inside button: `flex items-center justify-between w-full`
- Selected state: themed active toggle from CARD_THEMES (e.g. `bg-yellow-100 border-yellow-300` for caffeine)
- Max 8 presets visible without scroll. If more than 8, show first 6 + "Show all" expansion

#### AI Text Input

- Single row: `Input` component, full-width, `h-10`
- Placeholder: "Search beverage..." (coffee tab) / "Search drink..." (alcohol tab)
- Right side: inline button with `Sparkles` icon (16px), positioned inside the input using relative/absolute positioning
- Idle: icon in muted foreground color
- Loading: icon replaced with `Loader2` spinner (16px, `animate-spin`)
- Success: fields populate, input retains the search text
- Error: toast with "Lookup failed. Try a different name or enter values manually."

#### Volume and Substance Fields

- Two fields side by side: `grid grid-cols-2 gap-3`
- Volume field: `Input type="number"`, `h-10`, label "Volume (ml)" above in `text-xs text-muted-foreground`
- Per-100ml field: `Input type="number"`, `h-10`, label "per 100ml" above in `text-xs text-muted-foreground`
- Per-100ml field unit suffix: "mg/100ml" for caffeine, "std/100ml" for alcohol — displayed as inline text after the input or as placeholder
- Per-100ml field visibility: **always visible**. Pre-filled when a preset is selected or AI returns. Editable for manual correction.
- Fields pre-fill from: preset selection (immediate), AI lookup (on response), or manual typing

#### Calculated Amount Display

- Full-width row below the fields
- Format: `text-sm font-medium` with theme color
- Coffee: "{amount} mg caffeine" (e.g. "525 mg caffeine")
- Alcohol: "{amount} standard drinks" (e.g. "1.5 standard drinks")
- Calculation: `Math.round((volumeMl / 100) * substancePer100ml)` for caffeine, `((volumeMl / 100) * substancePer100ml).toFixed(1)` for alcohol
- Shows as soon as both volume and per-100ml fields have values > 0
- If either field is empty or zero: show "Enter volume and concentration" in `text-muted-foreground text-sm`

#### Action Buttons

- Two buttons side by side: `grid grid-cols-2 gap-3`
- "Log" button: `Button variant="outline"`, h-12, themed border color, full-width. Logs intake + substance record, does not save preset.
- "Save & Log" button: `Button variant="default"`, h-12, themed fill color, full-width. Saves as preset then logs. Shows confirmation toast: "Saved {name} as preset and logged."
- Both disabled when: `isSubmitting || volumeMl <= 0 || substancePer100ml <= 0`
- "Save & Log" additionally disabled when: text input is empty (no name to save)
- Loading state: button text changes to "Logging..." / "Saving..."

## Interaction States

### Tab Switching

| State | Behavior |
|-------|----------|
| Default tab | Water (D-02: always default on mount, no persistence) |
| Switch animation | `transition-all duration-300` on card gradient wrapper. No tab content animation (instant swap via Radix). |
| In-progress input | Preserved per tab. Each tab maintains its own local state. Switching away and back retains field values (React state not unmounted — use `forceMount` on TabsContent or keep state in parent). |
| Theme transition | Card gradient, border, icon, and button colors update to match active tab's CARD_THEMES entry |

### Preset Selection (Coffee/Alcohol tabs)

| State | Visual |
|-------|--------|
| Unselected | `Button variant="outline"` default styling |
| Selected | Themed active toggle: e.g. `bg-yellow-100 border-yellow-300 dark:bg-yellow-900/50 dark:border-yellow-700` |
| Tap effect | Fills volume and per-100ml fields. Text input cleared. Calculated amount updates immediately. |
| Re-tap same | Deselects: clears volume, per-100ml, and calculated amount |

### AI Lookup (Coffee/Alcohol tabs)

| State | Visual |
|-------|--------|
| Idle | Sparkles icon in `text-muted-foreground`, input enabled |
| Triggered | Input disabled, Sparkles replaced by `Loader2 animate-spin`, any selected preset deselected |
| Success | Input re-enabled, icon returns to Sparkles, per-100ml field populated from AI response, volume field populated with AI-suggested default volume, calculated amount updates |
| Error | Input re-enabled, icon returns to Sparkles, destructive toast: "Lookup failed. Try a different name or enter values manually." Fields unchanged. |
| Empty input + tap AI | No-op. Button visually disabled when input is empty. |

### Water Tab States (preserved from IntakeCard)

| State | Visual |
|-------|--------|
| Default | Pending amount = waterIncrement setting, +/- buttons enabled |
| At minimum | Minus button disabled (pendingAmount <= increment) |
| Would exceed limit | Pending amount text turns `text-orange-600 dark:text-orange-400` |
| Over limit | Progress bar turns `bg-red-500`, daily total turns `text-red-600 dark:text-red-400` |
| Submitting | Confirm button disabled, text "Recording..." |
| Center tap | Opens ManualInputDialog |

### Beverage Tab States

| State | Visual |
|-------|--------|
| Default | Empty name field, pending amount = waterIncrement setting |
| Name entered | Log button remains enabled (name is optional for logging, just tags the record) |
| Submitting | Button disabled, text "Logging..." |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Tab navigation | Radix Tabs handles arrow key navigation between tab triggers natively |
| Tab ARIA | Radix provides `role="tablist"`, `role="tab"`, `role="tabpanel"`, `aria-selected` automatically |
| Preset grid | Each preset button has accessible name from button text content |
| AI input | `aria-label="Search beverages for AI lookup"` on the input |
| AI button | `aria-label="Look up substance content"` on the sparkle icon button |
| Loading states | Buttons use `disabled` attribute; spinner has `aria-hidden="true"` |
| Volume/substance fields | Associated `Label` components with `htmlFor` |
| Color contrast | All text meets WCAG 2.1 AA (existing theme tokens verified in v1.0) |
| Reduced motion | Card gradient transition respects `prefers-reduced-motion` via existing CSS media query |
