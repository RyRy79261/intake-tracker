## Component Inventory

This phase introduces one new UI component and one new hook with UI side effects. All other work is service-layer (no visual output).

### New: TimezoneChangeDialog

**Type:** AlertDialog (shadcn `alert-dialog` component -- already installed)
**Trigger:** Programmatic open via `useTimezoneDetection` hook when IANA timezone name changes across app open/resume (per D-04, D-05)
**Dismissal:** User taps "Not Now" to dismiss; dialog does not reappear until next app open/resume (per D-07)

**Layout:**

```
+------------------------------------------+
|  [Globe icon]                            |
|                                          |
|  Timezone Changed                        |
|  It looks like you've traveled from      |
|  {oldTz} to {newTz}.                     |
|                                          |
|  +------------------------------------+  |
|  | Your dose times will stay at the   |  |
|  | same wall-clock times (e.g. 08:00  |  |
|  | stays 08:00) in your new timezone. |  |
|  +------------------------------------+  |
|                                          |
|  [Not Now]            [Adjust Schedules] |
+------------------------------------------+
```

**Anatomy:**

| Element | Component | Class / Style |
|---------|-----------|---------------|
| Icon | `Globe` from lucide-react | `w-6 h-6 text-medication` (teal, matches medication domain) |
| Title | `AlertDialogTitle` | `text-lg font-semibold` (inherited from shadcn) |
| Description | `AlertDialogDescription` | `text-sm text-muted-foreground` (inherited from shadcn) |
| Info box | `div` | `rounded-lg border bg-muted/50 p-3 text-sm text-muted-foreground mt-3` |
| Cancel button | `AlertDialogCancel` | `buttonVariants({ variant: "outline" })` -- "Not Now" |
| Confirm button | `AlertDialogAction` | `buttonVariants()` (primary default) -- "Adjust Schedules" |

**States:**

| State | Behavior |
|-------|----------|
| Open | Dialog renders when `useTimezoneDetection` detects cross-region change |
| Confirm | Calls bulk schedule recalculation, closes dialog, shows success toast |
| Dismiss | Closes dialog, stores session-level flag to prevent re-prompt until next app open/resume |
| Loading | Confirm button shows inline spinner (`Loader2` from lucide-react, `w-4 h-4 animate-spin`) while recalculation runs |

### New: useTimezoneDetection hook

**UI side effects:**
- Opens `TimezoneChangeDialog` when cross-region timezone change detected
- Shows info toast after silent DST adjustment (per D-06, Claude's discretion)
- Must be mounted once at app root level (in `providers.tsx` or medications layout)

### Existing: Toast (for success/DST feedback)

Reuses existing `toast()` from `@/hooks/use-toast`. No new toast component needed.

| Element | Copy |

| Dialog description | "It looks like you've traveled from {oldTimezone} to {newTimezone}." |

| Empty state heading | N/A -- this phase has no list/collection views |
| Empty state body | N/A |
| Error state | "Schedule adjustment failed. Your dose times have not changed. Try reopening the app." |

### Dynamic copy tokens

| Token | Source | Example |
|-------|--------|---------|
| `{oldTimezone}` | `PhaseSchedule.anchorTimezone` (human-readable via city name extraction) | "Johannesburg" |
| `{newTimezone}` | `Intl.DateTimeFormat().resolvedOptions().timeZone` (human-readable via city name extraction) | "Berlin" |

**City name extraction:** Strip the region prefix from IANA timezone names. `"Africa/Johannesburg"` becomes `"Johannesburg"`. `"Europe/Berlin"` becomes `"Berlin"`. Use `tz.split("/").pop()?.replace(/_/g, " ")` for display.

## Interaction Contract

### Timezone Change Detection Flow

```
App opens / resumes from background
    |
    v
useTimezoneDetection compares device IANA tz vs stored anchorTimezone
    |
    +-- Same IANA name, offset changed (DST) --> silent recalc + info toast (D-06)
    |
    +-- Different IANA name --> open TimezoneChangeDialog (D-05)
    |
    +-- No change --> no action
```

### Dialog Interaction

| Action | Result |
|--------|--------|
| Tap "Adjust Schedules" | 1. Button enters loading state (spinner). 2. Bulk recalculation updates all active PhaseSchedule records atomically. 3. Dialog closes. 4. Success toast appears. 5. Push schedule re-syncs automatically via usePushScheduleSync. |
| Tap "Not Now" | 1. Dialog closes. 2. Session flag set (Zustand non-persisted or module-level variable). 3. No schedule changes. 4. Dialog does not reappear until next app open/resume. |
| Tap overlay / press Escape | Same as "Not Now" (AlertDialog does not close on overlay tap by default -- this is correct behavior for a confirmation dialog). |

### Touch targets

All interactive elements meet 44px minimum touch target (AlertDialogAction and AlertDialogCancel both use `h-10` (40px) with additional padding, totaling > 44px effective tap area).
