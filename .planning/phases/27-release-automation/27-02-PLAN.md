# Plan 27-02: Commitlint & Husky Commit Hook

<objective>
Install commitlint and husky, configure conventional commit enforcement via a commit-msg hook so non-conventional commit messages are rejected before the commit is created.
</objective>

<threat_model>
## Threat Model

| ID | Threat | Severity | Mitigation |
|----|--------|----------|------------|
| T-27-02-01 | Hook not firing silently allows bad commits | Medium | Verify hook execution with explicit test: `git commit -m "bad" --allow-empty` must fail |
| T-27-02-02 | pnpm dlx resolution fails in hook context | Low | Use `pnpm dlx commitlint` which resolves from local devDependencies first; packages installed via `pnpm add -D` |
| T-27-02-03 | Husky hooks break CI runs (CI has no .git/hooks) | Low | Husky v9 is a no-op when `.git` directory is missing or `CI=true`; prepare script exits gracefully |

No high-severity threats. All mitigations are implemented in task actions.
</threat_model>

## Tasks

<task id="27-02-01">
<title>Install commitlint and husky packages</title>

<read_first>
- `package.json` (current devDependencies and scripts sections)
</read_first>

<action>
Install the three required packages as dev dependencies:

```bash
pnpm add -D @commitlint/cli @commitlint/config-conventional husky
```

Then initialize husky:

```bash
pnpm husky init
```

This will:
1. Add `@commitlint/cli`, `@commitlint/config-conventional`, and `husky` to `devDependencies` in `package.json`
2. Create the `.husky/` directory
3. Add `"prepare": "husky"` to the `scripts` section of `package.json`
4. Create a default `.husky/pre-commit` file (which we will leave as-is or remove if empty)

**Note:** The `preinstall` script (`npx only-allow pnpm`) already exists and must be preserved. The `prepare` script is a different lifecycle hook and does not conflict.
</action>

<acceptance_criteria>
- `package.json` devDependencies contains `@commitlint/cli`
- `package.json` devDependencies contains `@commitlint/config-conventional`
- `package.json` devDependencies contains `husky`
- `package.json` scripts contains `"prepare": "husky"`
- `.husky/` directory exists
- `pnpm dlx commitlint --version` outputs a version number (package is resolvable)
</acceptance_criteria>
</task>

<task id="27-02-02">
<title>Create commitlint configuration</title>

<read_first>
- `.planning/phases/27-release-automation/27-RESEARCH.md` (section 2.1 for config format)
- `package.json` (check `"type"` field — if `"module"`, use ESM export syntax)
</read_first>

<action>
Create `commitlint.config.js` at the repo root with this exact content:

```js
export default { extends: ['@commitlint/config-conventional'] };
```

This uses the `@commitlint/config-conventional` preset which:
- Enforces conventional commit format: `type(scope): description`
- Allows types: feat, fix, docs, style, refactor, perf, test, build, ci, chore, revert
- Auto-ignores merge commits (built into the preset's `ignores` rules)

**Note:** Use ESM `export default` syntax because the project likely has `"type": "module"` in package.json or uses `.js` extension with ESM. If the project uses CommonJS, use `module.exports = { extends: ['@commitlint/config-conventional'] };` instead. Check the `package.json` `"type"` field first.
</action>

<acceptance_criteria>
- File `commitlint.config.js` exists at repo root
- File contains `extends: ['@commitlint/config-conventional']`
- `echo "feat: test message" | pnpm dlx commitlint` exits with code 0
- `echo "invalid message" | pnpm dlx commitlint` exits with non-zero code
</acceptance_criteria>
</task>

<task id="27-02-03">
<title>Create commit-msg husky hook</title>

<read_first>
- `.husky/` directory contents (check what `husky init` created)
- `.planning/phases/27-release-automation/27-RESEARCH.md` (section 2.1 for hook content)
</read_first>

<action>
Create (or overwrite) `.husky/commit-msg` with this exact content:

```bash
pnpm dlx commitlint --edit $1
```

This file should:
- NOT have a shebang line (husky v9 handles shell invocation)
- Be executable (`chmod +x .husky/commit-msg`)
- Use `pnpm dlx commitlint` (not `npx`) for pnpm compatibility

After creating the file, verify the hook works:

```bash
# Should fail (non-conventional message)
git commit --allow-empty -m "bad message" 2>&1 && echo "FAIL: should have rejected" || echo "PASS: rejected bad message"

# Should succeed (conventional message)  
git commit --allow-empty -m "test: verify commitlint hook" && echo "PASS: accepted good message" || echo "FAIL: should have accepted"
```

If the test commit succeeds, amend it away or reset:
```bash
git reset HEAD~1
```
</action>

<acceptance_criteria>
- File `.husky/commit-msg` exists
- File contains `pnpm dlx commitlint --edit $1`
- `git commit --allow-empty -m "bad message"` is rejected (exit code non-zero)
- `echo "feat: test" | pnpm dlx commitlint` exits with code 0
- `echo "random text" | pnpm dlx commitlint` exits with non-zero code
</acceptance_criteria>
</task>

## Verification

<verification>
```bash
# Commitlint rejects bad messages
echo "bad message" | pnpm dlx commitlint && echo "FAIL" || echo "PASS"

# Commitlint accepts conventional messages
echo "feat: add feature" | pnpm dlx commitlint && echo "PASS" || echo "FAIL"

# Commitlint accepts all standard types
for type in feat fix docs style refactor perf test build ci chore revert; do
  echo "${type}: test" | pnpm dlx commitlint || echo "FAIL: ${type}"
done

# Husky hook file exists
test -f .husky/commit-msg && echo "PASS" || echo "FAIL"

# Config file exists
test -f commitlint.config.js && echo "PASS" || echo "FAIL"

# Packages installed
grep -q '"@commitlint/cli"' package.json && echo "PASS" || echo "FAIL"
grep -q '"husky"' package.json && echo "PASS" || echo "FAIL"
```
</verification>

<must_haves>
- commitlint rejects non-conventional commit messages
- commitlint accepts all standard conventional commit types
- Husky commit-msg hook calls commitlint on every commit
- Merge commits are not rejected by commitlint
</must_haves>
