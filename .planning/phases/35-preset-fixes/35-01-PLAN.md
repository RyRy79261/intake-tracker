# Plan 35-01: Gate save-as-preset button on AI lookup

<objective>
Prevent users from saving a new preset without first running AI substance lookup. The "Save as preset & log" button must be disabled (with a helper text explaining why) until the AI lookup has been performed and returned data.
</objective>

<context>
The `aiLookupUsed` state flag already exists in `preset-tab.tsx` and is set to `true` when AI lookup succeeds (line 141). It resets on `resetFields()`. Currently the "Save as preset & log" button is only gated on `isSubmitting || volumeMl <= 0 || !hasSubstance` but NOT on `aiLookupUsed`. This means users can manually type substance values and save a preset with inaccurate data.
</context>

## Tasks

### Task 1: Add AI lookup gate to save-as-preset button disabled condition

<read_first>
- `src/components/liquids/preset-tab.tsx` — Current disabled condition on lines 454-457, aiLookupUsed flag on line 30
- `.planning/phases/35-preset-fixes/35-UI-SPEC.md` — Component spec for disabled state
</read_first>

<action>
In `src/components/liquids/preset-tab.tsx`, modify the "Save as preset & log" button's `disabled` prop to include `!aiLookupUsed`:

Current disabled condition (lines 454-457):
```tsx
disabled={
  isSubmitting ||
  volumeMl <= 0 ||
  !hasSubstance
}
```

Change to:
```tsx
disabled={
  isSubmitting ||
  volumeMl <= 0 ||
  !hasSubstance ||
  !aiLookupUsed
}
```

This is a single prop addition. The `aiLookupUsed` state is already declared on line 30 and set to `true` in the `handleAiLookup` success path (line 141).
</action>

<acceptance_criteria>
- `preset-tab.tsx` contains the string `!aiLookupUsed` inside the disabled prop of the "Save as preset & log" button
- `grep -c "!aiLookupUsed" src/components/liquids/preset-tab.tsx` returns at least 1
- `pnpm tsc --noEmit` exits 0
</acceptance_criteria>

### Task 2: Add helper text below disabled save-as-preset button

<read_first>
- `src/components/liquids/preset-tab.tsx` — Current button rendering block (lines 449-463)
- `.planning/phases/35-preset-fixes/35-UI-SPEC.md` — Helper text spec: 12px text-xs, text-muted-foreground, "Use AI lookup to populate substance data"
</read_first>

<action>
In `src/components/liquids/preset-tab.tsx`, add helper text below the "Save as preset & log" button that shows when `!aiLookupUsed && beverageName.trim()`. Insert immediately after the closing `</Button>` of the save-as-preset button (before the closing `)}` of the conditional block):

```tsx
{!aiLookupUsed && beverageName.trim() && (
  <p className="text-xs text-muted-foreground text-center">
    Use AI lookup to populate substance data
  </p>
)}
```

This text appears only when:
1. AI lookup has NOT been used yet (`!aiLookupUsed`)
2. The user has typed a beverage name (`beverageName.trim()` is truthy)

It guides the user to use the sparkle/AI lookup before saving a preset.
</action>

<acceptance_criteria>
- `preset-tab.tsx` contains the string `Use AI lookup to populate substance data`
- The helper text is wrapped in a `<p>` tag with classes `text-xs text-muted-foreground text-center`
- The helper text conditionally renders only when `!aiLookupUsed && beverageName.trim()`
- `pnpm tsc --noEmit` exits 0
- `pnpm lint` exits 0
</acceptance_criteria>
