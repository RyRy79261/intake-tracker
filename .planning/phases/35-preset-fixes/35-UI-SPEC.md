## Component Specifications

### 1. "Save as preset & log" Button â€” Disabled State (PRES-01)

**Current state:** Button appears when `beverageName.trim()` is truthy, no AI gate.

**Target state:**

| Property | Value |
|----------|-------|
| Visibility | Visible whenever `beverageName.trim()` is truthy (unchanged) |
| Disabled condition | `isSubmitting \|\| volumeMl <= 0 \|\| !hasSubstance \|\| !aiLookupUsed` |
| Disabled appearance | `opacity-50 cursor-not-allowed` (shadcn Button default disabled style) |
| Helper text | 12px (text-xs), `text-muted-foreground`, below the button: "Use AI lookup to populate substance data" |
| Helper visibility | Only shown when `!aiLookupUsed && beverageName.trim()` |
| Enabled appearance | Standard outline button with tab-specific `theme.outlineBorder` (unchanged) |

**Copywriting:**
- Button label (enabled): "Save as preset & log" (unchanged)
- Button label (disabled): "Save as preset & log" (same text, just visually disabled)
- Helper text: "Use AI lookup to populate substance data"

### 2. Preset Grid Long-Press Delete (PRES-02)

**Gesture interaction:**

| Property | Value |
|----------|-------|
| Trigger | `onPointerDown` starts 500ms timer |
| Cancel events | `onPointerUp`, `onPointerCancel`, `onPointerLeave` clear timer |
| CSS | `touch-action: manipulation` on preset buttons (prevents browser zoom on long-press) |
| Click prevention | `longPressTriggered` ref prevents `onClick` from firing after long-press |
| Visual feedback | None during press (keep simple; no scale/opacity animation) |

**Confirm Dialog (AlertDialog):**

| Property | Value |
|----------|-------|
| Component | shadcn AlertDialog (Radix `@radix-ui/react-alert-dialog`) |
| Controlled by | `deletePresetId: string \| null` state |
| Open condition | `deletePresetId !== null` |
| Overlay | `bg-black/80` (shadcn default) |
| Content | `max-w-lg`, `p-6`, centered (shadcn default) |
| Title | `text-lg font-semibold`: "Delete {preset name}?" |
| Description | `text-sm text-muted-foreground`: "This preset will be permanently removed." |
| Cancel button | shadcn `AlertDialogCancel` â€” outline variant, label: "Cancel" |
| Delete button | shadcn `AlertDialogAction` with `bg-destructive text-destructive-foreground hover:bg-destructive/90`, label: "Delete" |
| Button order | Cancel left, Delete right (desktop), Delete bottom, Cancel top (mobile â€” shadcn `flex-col-reverse`) |

**After deletion:**
- Zustand `deleteLiquidPreset(id)` removes preset from array
- React re-render removes button from grid immediately
- Toast notification: title "Deleted", description "{preset name} removed", variant "default"
