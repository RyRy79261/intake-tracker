  <name>Task 1: Create query registry with Zod parameter schemas + registry tests</name>
  <files>src/lib/analytics-registry.ts, src/lib/analytics-registry.test.ts</files>
  <action>
Create `src/lib/analytics-registry.ts`:

1. Define `QueryDescriptor` interface:
```typescript
export interface QueryDescriptor {
  id: string;
  name: string;
  description: string;
  category: 'fluid' | 'medication' | 'vitals' | 'correlation' | 'custom';
  parameters: z.ZodSchema;
  execute: (params: Record<string, unknown>) => Promise<AnalyticsResult<unknown>>;
}
```

2. Define Zod parameter schemas:
   - `TimeRangeSchema`: `z.object({ start: z.number(), end: z.number() })`
   - `AdherenceParamsSchema`: `z.object({ start: z.number(), end: z.number(), prescriptionId: z.string().optional() })`
   - `CorrelationParamsSchema`: `z.object({ start: z.number(), end: z.number(), domainA: z.enum([...domains]), domainB: z.enum([...domains]), lagDays: z.number().optional() })`
   - `SaltWeightParamsSchema`: `z.object({ start: z.number(), end: z.number(), lagDays: z.number().optional() })`

3. Export `queryRegistry: QueryDescriptor[]` with entries for each pre-built query:
   - `fluid_balance` -- Fluid Balance, fluid category, TimeRangeSchema
   - `adherence_rate` -- Medication Adherence, medication category, AdherenceParamsSchema
   - `bp_trend` -- Blood Pressure Trend, vitals category, TimeRangeSchema
   - `weight_trend` -- Weight Trend, vitals category, TimeRangeSchema
   - `salt_vs_weight` -- Salt vs Weight Correlation, correlation category, SaltWeightParamsSchema
   - `caffeine_vs_bp` -- Caffeine vs BP Correlation, correlation category, TimeRangeSchema
   - `alcohol_vs_bp` -- Alcohol vs BP Correlation, correlation category, TimeRangeSchema
   - `custom_correlation` -- Custom Domain Correlation, custom category, CorrelationParamsSchema

4. Each entry's `execute` function: parse params through the Zod schema, call the corresponding analytics service function, return result.

5. Export helper functions:
   - `getQueryById(id: string): QueryDescriptor | undefined`
   - `listQueries(): Array<{ id: string; name: string; description: string; category: string }>`

Create `src/lib/analytics-registry.test.ts`:
   - Test: queryRegistry has 8 entries (fluid_balance, adherence_rate, bp_trend, weight_trend, salt_vs_weight, caffeine_vs_bp, alcohol_vs_bp, custom_correlation)
   - Test: each entry has valid Zod schema (can parse valid params without throwing)
   - Test: getQueryById returns correct entry for each ID
   - Test: getQueryById returns undefined for unknown ID
   - Test: listQueries returns array with id, name, description, category for each entry
   - Test: every query has a non-empty description (AI agent needs meaningful descriptions)

<task type="auto">
  <name>Task 2: Create analytics hooks with useLiveQuery</name>
  <files>src/hooks/use-analytics-queries.ts</files>
  <action>
Create `src/hooks/use-analytics-queries.ts` with "use client" directive:

1. Import useLiveQuery from dexie-react-hooks, analytics service functions, and types.

2. Define default values for each result type (eliminates loading states per Phase 3 pattern):
   - `DEFAULT_FLUID_BALANCE`: AnalyticsResult with empty FluidBalanceResult
   - `DEFAULT_ADHERENCE`: AnalyticsResult with rate=0, taken=0, total=0
   - `DEFAULT_BP_TREND`: AnalyticsResult with empty readings, stable trends
   - `DEFAULT_WEIGHT_TREND`: AnalyticsResult with empty readings, stable trend
   - `DEFAULT_CORRELATION`: AnalyticsResult with coefficient=0, strength='none'

3. Create hooks:
   - `useFluidBalance(range: TimeRange)` -- useLiveQuery calling fluidBalance, deps [range.start, range.end], default DEFAULT_FLUID_BALANCE
   - `useAdherenceRate(range: TimeRange, prescriptionId?: string)` -- useLiveQuery calling adherenceRate, deps [range.start, range.end, prescriptionId]
   - `useBPTrend(range: TimeRange)` -- useLiveQuery calling bpTrend
   - `useWeightTrend(range: TimeRange)` -- useLiveQuery calling weightTrend
   - `useSaltVsWeight(range: TimeRange, lagDays?: number)` -- useLiveQuery calling saltVsWeight
   - `useCaffeineVsBP(range: TimeRange)` -- useLiveQuery calling caffeineVsBP
   - `useAlcoholVsBP(range: TimeRange)` -- useLiveQuery calling alcoholVsBP
   - `useCorrelation(domainA: Domain, domainB: Domain, range: TimeRange, lagDays?: number)` -- useLiveQuery calling correlate from analytics-service, returns CorrelationResult

4. Create `useInsights(range: TimeRange)` hook:
   - Calls multiple analytics queries (adherenceRate, bpTrend, fluidBalance) inside a single useLiveQuery callback
   - Derives insights from results: adherence drops below 80%, BP trending up/down significantly, fluid deficit days
   - Returns `Insight[]` with default `[]`
   - Insight generation logic: check adherenceRate < 0.8 (adherence_drop), check bpTrend slope significance (bp_trend), check fluid deficit ratio (fluid_deficit), check anomalies via detectAnomalies on weight (anomaly)

5. Create `useTimeScopeRange(scope: TimeScope): TimeRange` utility hook:
   - Converts TimeScope ('24h', '7d', '30d', '90d', 'all') to TimeRange { start, end }
   - '24h': last 24 hours, '7d': last 7 days, etc.
   - 'all': start=0, end=Date.now()
   - Memoized with useMemo to prevent unnecessary re-renders

6. All hooks follow established pattern: useLiveQuery with default values, deps arrays contain all varying parameters.
