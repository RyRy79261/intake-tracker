1. Add `SubstanceRecord` interface to db.ts:
```typescript
export interface SubstanceRecord {
  id: string;
  type: 'caffeine' | 'alcohol';
  amountMg?: number;           // caffeine mg
  amountStandardDrinks?: number; // alcohol standard drinks
  volumeMl?: number;            // liquid volume for fluid balance linking
  description: string;
  source: 'water_intake' | 'eating' | 'standalone';
  sourceRecordId?: string;      // links to original intake record if migrated
  timestamp: number;
  createdAt: number;
  updatedAt: number;
  deletedAt: number | null;
  deviceId: string;
  timezone: string;
}
```

2. Add `substanceRecords` EntityTable to db type declaration.

3. Add `db.version(12).stores({...})` -- MUST repeat ALL v11 store definitions plus add:
   `substanceRecords: "id, [type+timestamp], type, timestamp, source, sourceRecordId, updatedAt"`

4. Add upgrade function with keyword extraction from existing intake records:
   - Define keyword maps: `CAFFEINE_KEYWORDS = ['coffee', 'espresso', 'tea', 'caffeine', 'matcha', 'latte', 'cappuccino']`
   - Define keyword maps: `ALCOHOL_KEYWORDS = ['beer', 'wine', 'whiskey', 'whisky', 'vodka', 'gin', 'rum', 'cocktail', 'spirit', 'alcohol', 'brandy']`
   - Define default amounts: `DEFAULT_CAFFEINE_MG = { coffee: 95, espresso: 63, tea: 47, latte: 95, cappuccino: 95, matcha: 70 }` (fallback: 95)
   - Define default volumes: `DEFAULT_CAFFEINE_VOLUME_ML = { coffee: 250, espresso: 30, tea: 250, latte: 350, cappuccino: 250, matcha: 250 }` (fallback: 250)
   - Define default alcohol: `DEFAULT_ALCOHOL_DRINKS = { beer: 1, wine: 1, cocktail: 1.5 }` (fallback: 1)
   - Scan ALL intakeRecords -- check `note` field (lowercased) for keyword matches
   - For each match: create a SubstanceRecord with `source: 'water_intake'`, `sourceRecordId: record.id`, correct `amountMg` or `amountStandardDrinks`, and use the record's existing timestamp/timezone
   - NO network calls inside migration (Pitfall 1 from research)
   - Use `crypto.randomUUID()` for IDs, backfill sync fields from original record

5. Add `makeSubstanceRecord()` fixture factory to `src/__tests__/fixtures/index.ts` following existing pattern.

1. Create `src/lib/substance-service.ts`:
   - `addSubstanceRecord(input: AddSubstanceInput): Promise<ServiceResult<SubstanceRecord>>` -- wraps in `db.transaction('rw', [db.substanceRecords, db.intakeRecords], ...)`. Creates substance record. If `volumeMl` provided, also creates a linked intakeRecord with `type: 'water'`, `source: 'substance:{substanceId}'`, `note: input.description`. Uses `syncFields()` for both records.
   - `getSubstanceRecords(type?: 'caffeine' | 'alcohol', limit?: number): Promise<SubstanceRecord[]>` -- optional type filter, ordered by timestamp desc
   - `getSubstanceRecordsByDateRange(startTime: number, endTime: number, type?: 'caffeine' | 'alcohol'): Promise<SubstanceRecord[]>` -- date range query with optional type filter. Use compound index `[type+timestamp]` when type provided, plain `timestamp` index otherwise.
   - `deleteSubstanceRecord(id: string): Promise<ServiceResult<void>>` -- soft delete (set deletedAt). Also soft-delete the linked intakeRecord if it has `source: 'substance:{id}'`
   - `updateSubstanceRecord(id: string, updates: Partial<SubstanceRecord>): Promise<ServiceResult<void>>` -- update with updatedAt timestamp
   - `AddSubstanceInput` type: `{ type: 'caffeine' | 'alcohol'; amountMg?: number; amountStandardDrinks?: number; volumeMl?: number; description: string; source?: 'standalone'; timestamp?: number }`
   - Default `source` to 'standalone', default `timestamp` to Date.now()
   - Use `ok()` / `err()` from service-result for return types
