## Interaction Contract

> Phase-specific section: defines the tap-to-type interaction behavior.

### Activation
- User taps the center value display area (the `text-4xl font-bold` number + "kg" suffix)
- The hidden `<input>` inside the wrapping `<label>` receives focus via native label association
- On mobile: numeric keyboard appears (`type="number"`, `inputMode="decimal"`)
- On desktop: text cursor appears in the value, keyboard input is accepted

### Visual During Input
- Display text updates in real-time as user types (reading from hidden input value)
- Format mask: `{value} kg` — the "kg" suffix remains visible at all times
- No borders, no outline, no ring, no background shift — the display looks identical to read mode except for the text cursor
- The `tabular-nums` font feature prevents digit-width jitter during typing

### Blur / Deactivation
- On blur (tap away or press Enter): typed value is rounded to nearest `settings.weightIncrement` (default 0.05)
- Rounded value replaces display text — user sees the corrected value
- No auto-save: user must press "Record Weight" button to persist

### Edge Cases
- Empty input on blur: revert to previous `pendingWeight` value
- Non-numeric input: prevented by `type="number"` + `inputMode="decimal"`
- Value outside range (<=0 or >1000): clamp to valid range on blur
- Stepper buttons (+/-) still work while input is not focused — no conflict

### Component Structure
- New reusable component in `src/components/ui/` (shadcn convention)
- Props: accepts any native input `type`, a display `format` function, `value`, `onChange`, `onBlur`
- The component wraps a visually-hidden `<input>` inside a `<label>` with the formatted display text
- Hidden input must be accessible: `getByRole('spinbutton')` or `data-testid` for Playwright
