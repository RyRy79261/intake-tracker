    - `enqueue(tableName: TableName, recordId: string, op: "upsert" | "delete"): Promise<void>` wrapped in a Dexie `rw` transaction on `db._syncQueue`
    - Coalesce rules (D-04):
      - No existing row → add new `{tableName, recordId, op, enqueuedAt: Date.now(), attempts: 0}`
      - Existing upsert + incoming upsert → update enqueuedAt only
      - Existing upsert + incoming delete → change op to "delete", reset attempts to 0, update enqueuedAt (delete supersedes)
      - Existing delete + incoming upsert → change op to "upsert", reset attempts, update enqueuedAt (un-delete)
      - Existing delete + incoming delete → update enqueuedAt only
    - `ack(queueIds: number[]): Promise<void>` deletes exactly those rows via `db._syncQueue.bulkDelete(queueIds)`
    - `getQueueDepth(): Promise<number>` returns `db._syncQueue.count()` — no transaction needed
    - `writeWithSync<T>(tableName, op, action): Promise<T>` wraps `action()` and the enqueue in a single `db.transaction("rw", db.table(tableName), db._syncQueue, async () => { ... })` — on throw, both are rolled back
    - When `action()` in writeWithSync returns a record with `id` field, that id is extracted for enqueue — the helper infers recordId from the returned record (`action: () => Promise<T extends { id: string }>`)
    - Every exported helper is pure-ish — no event emitters, no network calls, no timers (those belong to Plan 06's engine)
  </behavior>

    5. Export `async function writeWithSync<T extends { id: string }>(tableName: TableName, op: "upsert" | "delete", action: () => Promise<T>): Promise<T>`:
       ```
       return db.transaction("rw", db.table(tableName), db._syncQueue, async () => {
         const record = await action();
         await enqueueInside(tableName, record.id, op);  // inner version that reuses outer tx
         return record;
       });
       ```
       Provide an internal `enqueueInside` or factor enqueue so Dexie does not nest transactions (Dexie throws if you open `rw` inside `rw` for overlapping tables; use the outer transaction's implicit scope — documented on the Dexie docs page linked in 43-RESEARCH.md Pattern 5).
