    - `runPushCycle()` is idempotent: if `pushInFlight` flag set OR `navigator.onLine === false`, returns immediately
    - Push cycle steps (per 43-RESEARCH.md Pattern 3):
      1. Read up to 200 queue rows (ordered by enqueuedAt ASC, grouped by tableName in TABLE_PUSH_ORDER sequence)
      2. `bulkGet` current Dexie row snapshots for upsert ops; delete ops carry the tombstone row
      3. POST /api/sync/push with body `{ ops }`
      4. On !res.ok → parse JSON if possible for diagnostic, increment attempts counter on all sent queue rows via `db._syncQueue.update`, call `schedulePush(nextBackoff(maxAttempts))`, update store with lastError
      5. On res.ok → parse `{ accepted }`, call `ack(accepted.map(a => a.queueId))`, call `applyServerAck(accepted)` which updates local row `updatedAt` only if `local.updatedAt <= server.updatedAt` (Pitfall 3), set store.lastPushedAt + clear lastError, schedule a pull
    - `runPullCycle()` iterates TABLE_PUSH_ORDER (order doesn't matter for pull — use the same for consistency), reads per-table cursor from `_syncMeta`, POSTs /api/sync/pull with the cursor map, applies response in a transaction, advances cursor with skew margin, re-calls while any table reports `hasMore: true`
    - Skew margin constant: `const SKEW_MARGIN_MS = 30_000;`
    - Debounce constant: `const DEBOUNCE_MS = 3000;`
    - Batch cap: `const PUSH_BATCH_CAP = 200;`
    - `startEngine()` is called by Plan 06 Task 2's lifecycle hook. It sets isOnline from navigator, kicks a startup pullCycle, and attaches `window.__syncEngine` dev hook if `process.env.NODE_ENV !== 'production'`. Calling startEngine twice is a no-op (guarded by module-level started flag).
    - Pull writes are atomic per-table: `db.transaction('rw', [db.table(tableName), db._syncMeta], async () => { bulkPut(rows); put({tableName, lastPulledUpdatedAt: newCursor}); })`
    - After a pull cycle completes, calls `queryClient.invalidateQueries()` — use the singleton QueryClient from wherever the app exports it (grep `queryClient` in src/ to find the module; likely `src/lib/query-client.ts` or inline in providers.tsx — if inline, Plan 06 must extract it to a shared module as part of Task 3).

    Recommended layout (~180-250 lines):

    1. Imports: `db` + `_syncQueue/_syncMeta` types from @/lib/db; `enqueue, ack, getQueueDepth` from @/lib/sync-queue; `TABLE_PUSH_ORDER, TableName` from @/lib/sync-topology; `PULL_SOFT_CAP, PullResponse, PushOp` from @/lib/sync-payload; `useSyncStatusStore` from @/stores/sync-status-store.

    2. Module constants: `DEBOUNCE_MS = 3000`, `PUSH_BATCH_CAP = 200`, `BACKOFF_BASE_MS = 2000`, `BACKOFF_CAP_MS = 60_000`, `JITTER_RATIO = 0.2`, `SKEW_MARGIN_MS = 30_000`.

    3. Module-local mutable state (not exported): `pushTimer: ReturnType<typeof setTimeout> | null = null;`, `pushInFlight = false;`, `pullInFlight = false;`, `engineStarted = false;`.

    4. `export function nextBackoff(attempts: number): number` — pure. `const base = Math.min(BACKOFF_BASE_MS * Math.pow(2, attempts), BACKOFF_CAP_MS); const jitter = 1 - JITTER_RATIO + Math.random() * (2 * JITTER_RATIO); return Math.round(base * jitter);`

    5. `export function schedulePush(delayMs: number = DEBOUNCE_MS)` — `if (pushTimer) clearTimeout(pushTimer); pushTimer = setTimeout(() => { pushTimer = null; void runPushCycle(); }, delayMs);`

    6. `export async function runPushCycle(): Promise<void>` — full body per behavior spec. Handles: in-flight guard, offline guard, collect+order ops, fetch, success/error branches, ack + applyServerAck, schedulePull on success, incrementAttempts + schedulePush(nextBackoff) on failure. Updates useSyncStatusStore via setState throughout.

    7. `async function collectAndOrderQueuedOps(): Promise<PushOp[]>` (internal) — reads up to PUSH_BATCH_CAP queue rows, iterates TABLE_PUSH_ORDER, bulkGets row snapshots for upserts, assembles the PushOp[] with `{queueId, tableName, op, row}`. For delete ops, reads the Dexie row including `deletedAt` — if the row was hard-deleted (unlikely per P43 scope) carry a stub `{id, deletedAt: Date.now(), ...}`.

    8. `async function applyServerAck(accepted: Array<{queueId: number; serverUpdatedAt: number}>): Promise<void>` (internal) — per Pitfall 3: for each accepted entry look up the original queue row (before ack-delete) to find `{tableName, recordId}`, then check `local = await db.table(tableName).get(recordId)`, if `local && local.updatedAt <= serverUpdatedAt` then `db.table(tableName).update(recordId, { updatedAt: serverUpdatedAt })`. Ack delete happens via `ack(queueIds)` separately.

    9. `export function schedulePull(): void` — enqueues a pull on the microtask queue to avoid synchronous recursion from schedulePush. `queueMicrotask(() => { void runPullCycle(); });`

    10. `export async function runPullCycle(): Promise<void>` — full body. Loop: build cursors map from _syncMeta (default 0 if missing), POST /api/sync/pull, on !res.ok update lastError and return, on res.ok iterate `result` per tableName, compute `const maxUpdatedAt = rows.length ? Math.max(...rows.map(r => r.updatedAt)) : cursor;` and `const safeCursor = Math.min(maxUpdatedAt, serverTime - SKEW_MARGIN_MS);`, open `db.transaction("rw", [db.table(tableName), db._syncMeta], async () => { await db.table(tableName).bulkPut(rows); await db._syncMeta.put({ tableName, lastPulledUpdatedAt: safeCursor }); })`, continue while any table had hasMore. After loop: `useSyncStatusStore.getState().markPulled();` + `getQueryClient().invalidateQueries();`.

    11. `export function startEngine(): void` — guard via `engineStarted`. Set initial `useSyncStatusStore.setState({ isOnline: navigator.onLine })`. Kick `schedulePull()` for startup pull. In NODE_ENV !== 'production', attach TWO dev-only window hooks: (a) `window.__syncEngine = { pushNow: () => runPushCycle(), pullNow: () => runPullCycle(), getQueueDepth }`, and (b) `window.__devIntake = { add: addIntakeRecord, getIntake: (id) => db.intakeRecords.get(id) }` — used by Plan 08's E2E spec to seed records and read them back without relying on dynamic TS imports. Import `addIntakeRecord` from `@/lib/intake-service` and `db` from `@/lib/db` at the top of the file.
