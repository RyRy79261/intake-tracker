**Step 1: Add security settings to pnpm-workspace.yaml.**

Add the following 4 settings AFTER the existing `onlyBuiltDependencies` block, with a comment header. The final file should look like:

```yaml
onlyBuiltDependencies:
  - '@reown/appkit'

# Supply chain security (Phase 23)
minimumReleaseAge: 1440
trustPolicy: no-downgrade
blockExoticSubdeps: true
auditLevel: high
```

Remove the existing `auditConfig: {}` line (it is empty/unused and will be replaced by the `auditLevel` setting).

Per D-03: No override mechanism for minimumReleaseAge. The 24h quarantine has no escape hatch.

**Step 2: Bump direct dependencies with known critical/high vulns.**

In package.json, update:
- `"next": "14.2.15"` to `"next": "14.2.35"` (pin exact version, not caret -- matches current pinning style)
- `"jspdf": "^4.2.0"` to `"^4.2.1"` (caret range, minimum 4.2.1 which fixes the PDF injection vuln)

**Step 3: Run pnpm install to update lockfile.**

Run `pnpm install` (NOT --frozen-lockfile, since we changed deps). This regenerates pnpm-lock.yaml with the bumped versions.

**Step 4: Fix transitive vulnerabilities.**

Run `pnpm audit --fix` to auto-generate `overrides` in pnpm-workspace.yaml for remaining transitive high/critical vulns (minimatch, serialize-javascript, rollup, brace-expansion, flatted, picomatch, etc. via next-pwa, eslint, @privy-io/react-auth). Review the generated overrides -- they should pin safe versions of vulnerable transitive deps.

After `pnpm audit --fix`, run `pnpm install` again to apply overrides.

**Step 5: Handle trustPolicy compatibility.**

After adding `trustPolicy: no-downgrade`, if `pnpm install` fails with trust-related errors for specific packages, add a `trustPolicyExclude` list in pnpm-workspace.yaml with only the specific packages that fail. Keep the exclusion list minimal. If no packages fail, do not add trustPolicyExclude.

**Step 6: Verify clean audit.**

Run `pnpm audit --audit-level high`. It MUST return exit code 0 (no critical/high findings). Per D-01, moderate/low findings are acceptable. Per D-02, there must be no allowlists or warn-only mode -- the audit gate must be meaningful from day one.

**Step 7: Verify build still works.**

Run `pnpm build` to confirm the dependency bumps (especially next@14.2.35) don't break the production build.

Per D-05: audit checks all dependencies (prod + dev). Do NOT use --prod flag.
