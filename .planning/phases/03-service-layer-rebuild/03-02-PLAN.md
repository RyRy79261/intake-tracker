<task type="auto">
  <name>Task 1: Rebuild dose-log-service with atomic transactions and fractional math</name>
  <files>src/lib/dose-log-service.ts</files>
  <action>
Rewrite `src/lib/dose-log-service.ts` completely. Key changes:

**Read functions return T directly (throw on error):**
- `getDoseLogsForDate(date: string): Promise<DoseLog[]>` ã no ServiceResult wrapper
- `getDoseLogsWithDetailsForDate(date: string): Promise<DoseLogWithDetails[]>` ã no ServiceResult wrapper
- `getDoseLog(...)` returns `Promise<DoseLog | undefined>`

**Mutation functions keep ServiceResult and are atomic:**

1. `takeDose(input: TakeDoseInput): Promise<ServiceResult<DoseLog>>` where TakeDoseInput is `{ prescriptionId, phaseId, scheduleId, date, time, dosageMg: number }`.
   - Wrap EVERYTHING in a single `db.transaction("rw", [db.doseLogs, db.inventoryItems, db.inventoryTransactions, db.auditLogs], async () => { ... })`
   - Inside the transaction:
     a. Check if dose already taken (skip stock adjustment if re-taking)
     b. Upsert dose log with status "taken", actionTimestamp = Date.now(), timezone = getDeviceTimezone()
     c. Find active inventory: `db.inventoryItems.where({ prescriptionId, isActive: 1 }).first()` ã if none found, log dose without inventory (inventoryItemId remains undefined). This is explicitly allowed per user decision.
     d. If inventory found: calculate `pillsConsumed = Math.round((dosageMg / inventory.strength) * 10000) / 10000`. Update `inventory.currentStock = (inventory.currentStock ?? 0) - pillsConsumed`. Allow negative stock (per user decision). Add inventoryTransaction with type "consumed", amount = -pillsConsumed, doseLogId = log.id.
     e. If `pillsConsumed` is not a clean fraction (not 0.25, 0.333, 0.5, 0.667, 0.75, or whole number), include a `warning: "odd_fraction"` in the audit details.
     f. Add audit log entry via `db.auditLogs.add(buildAuditEntry("dose_taken", { prescriptionId, date, time, dosageMg, pillsConsumed, inventoryItemId }))`
   - Do NOT call the standalone `adjustStock` function ã inline the stock math to avoid nested transaction issues (Pitfall 4 from research).

2. `untakeDose(input: UntakeDoseInput)` ã same pattern. Reverse the stock: add pillsConsumed back. Add "consumed" transaction with positive amount. Audit action "dose_untaken". Set dose log status back to "pending".

3. `skipDose(input: SkipDoseInput)` ã if previously taken, reverse stock. Set status "skipped". Audit action "dose_skipped".

4. `rescheduleDose(input: RescheduleDoseInput)` ã if previously taken, reverse stock on old time. Mark old slot "rescheduled", create new slot "pending". Audit action "dose_rescheduled".

5. `takeAllDoses` / `skipAllDoses` ã each individual dose gets its own transaction (per user decision: "individual transactions per dose, one failure doesn't block others"). Collect results, return aggregate ServiceResult.

**Remove `generatePendingDoseLogs`** ã dose generation will be derive-at-read in Plan 04's daily schedule service.

**Helper: `calculatePillsConsumed(doseMg: number, pillStrengthMg: number): number`** ã exported for reuse. Uses `Math.round(raw * 10000) / 10000`.

**Helper: `isCleanFraction(pillsConsumed: number): boolean`** ã exported. Checks if fractional part is 0, 0.25, 0.333, 0.5, 0.667, or 0.75 (within 0.01 tolerance).
