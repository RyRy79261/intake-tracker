Create `src/components/medications/phase-timeline.tsx` with the following structure:

**Exported component: `PhaseTimeline`**
- Props: `{ phases: MedicationPhase[]; prescriptionId: string; currentUnit: string }`
- Sort phases by `startDate` descending (future/planned at top, completed at bottom). For phases without startDate or same startDate, use status tiebreaker: pending > active > completed > cancelled.
- If zero phases: return null (hide section entirely per Claude's discretion)
- If exactly one phase: render just the phase detail (expanded) without timeline connecting line or dots -- no timeline chrome for single-phase prescriptions
- If multiple phases: render vertical timeline with connecting line

**Timeline layout (multiple phases):**
- Container: `relative pl-6` with absolute vertical line `left-[5px] top-3 bottom-3 w-0.5 bg-border`
- Each phase node positioned relative to the line
- Between adjacent nodes (index > 0), render a `TransitionLabel`

**Sub-component: `TimelineDot`**
- Props: `{ status: MedicationPhase["status"] }`
- Completed: filled dot -- `bg-muted-foreground border-muted-foreground` (solid filled circle)
- Active: ring dot -- `bg-background border-emerald-500 ring-2 ring-emerald-500/30`
- Pending: empty dot -- `bg-background border-muted-foreground/50`
- Cancelled: dimmed -- `bg-muted border-muted-foreground/30`
- All dots: `w-3 h-3 rounded-full border-2 flex-shrink-0`

**Sub-component: `TimelineNode`**
- Props: `{ phase: MedicationPhase; isActive: boolean; defaultExpanded: boolean }`
- Each `TimelineNode` calls `useSchedulesForPhase(phase.id)` internally (avoids hooks-in-loop violation)
- Local `useState` for `isExpanded`, initialized to `defaultExpanded`
- Active phase: always starts expanded (`defaultExpanded=true`), has green left border: `border-l-2 border-l-emerald-500`
- Non-active phases: start collapsed, tap anywhere on the node to toggle expand
- Cancelled phases: `opacity-50`
- Active phase gets a ref for auto-scroll (passed via `activeRef` prop)

**Sub-component: `PhaseDetailCompact`**
- Shows: phase name/type badge + one-liner summary
- For completed: "{totalDosage}{unit} . {startDateFormatted} - {endDateFormatted} . {durationStr}"
- For pending: "{totalDosage}{unit} . starts {startDateFormatted}"
- Type badge: blue for maintenance ("Maintenance"), amber for titration ("Titration") -- same colors as TitrationPhaseCard
- Date format: `new Date(ts).toLocaleDateString(undefined, { month: "short", day: "numeric" })`
- Duration: days >= 7 ? `${Math.round(days / 7)} weeks` : `${days} days`
- Handle empty schedules (show "No dosage" instead of "0mg")

**Sub-component: `PhaseDetailExpanded`**
- Shows: type badge + status badge + full schedule list (time + dosage per enabled schedule entry) + date range + duration + food instruction
- Schedule entries sorted by `time` ascending
- Each schedule line: "{time} - {dosage}{unit}"
- Food instruction: "Take before eating" / "Take after eating" / omitted for "none"
- Date range: "{startDate} - {endDate or 'present'}"
- Duration line below date range

**Sub-component: `TransitionLabel`**
- Own component that calls `useSchedulesForPhase` for both fromPhaseId and toPhaseId
- Compute total enabled dosage for each phase
- If dosage differs: show arrow with change, e.g. `\u25B2 75mg \u2192 150mg` (up arrow for increase) or `\u25BC 150mg \u2192 75mg` (down arrow for decrease)
- If dosage same but schedules differ: show "Schedule changed"
- If both same: show nothing (return null)
- Styled: `text-[10px] text-muted-foreground py-1 pl-6` positioned on the timeline line area

**Auto-scroll:**
- `useRef<HTMLDivElement>(null)` for active phase node
- `useEffect` with 300ms setTimeout to `scrollIntoView({ behavior: "smooth", block: "center" })`

**Animations:**
- Use `motion/react` (NOT `framer-motion`) for expand/collapse: `AnimatePresence` wrapping expanded content
- `initial={{ height: 0, opacity: 0 }}`, `animate={{ height: "auto", opacity: 1 }}`, `exit={{ height: 0, opacity: 0 }}`, `transition={{ duration: 0.2 }}`
- `overflow-hidden` on the motion.div

**Import from project:**
- `import { motion, AnimatePresence } from "motion/react"`
- `import { Badge } from "@/components/ui/badge"`
- `import { useSchedulesForPhase } from "@/hooks/use-medication-queries"`
- `import type { MedicationPhase } from "@/lib/db"`
- `import { cn } from "@/lib/utils"`

Do NOT import or reuse `TitrationPhaseCard` -- this component replaces its usage in the drawer context.
