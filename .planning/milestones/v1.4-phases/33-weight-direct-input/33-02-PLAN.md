     <span className="text-4xl font-bold tabular-nums">

   Replace with:
   ```tsx
   {/* Center Display — tap to type (D-01, D-02) */}
   <div className="flex-1 text-center">
     <InlineEdit
       value={pendingWeight}
       onValueChange={setPendingWeight}
       formatDisplay={(v) => v?.toFixed(2) ?? "--"}
       suffix="kg"
       displayClassName="text-4xl font-bold tabular-nums"
       suffixClassName="text-lg text-muted-foreground ml-1"
       roundOnBlur={(v) => {
         const increment = settings.weightIncrement;
         const rounded = Math.round(v / increment) * increment;
         return Math.round(rounded * 100) / 100;
       }}
       type="number"
       inputMode="decimal"
       step="any"
       min={0.1}
       max={1000}
       aria-label="Weight in kilograms"
       data-testid="weight-direct-input"
     />
   </div>
   ```

3. **No other changes** — the existing `handleSubmit`, `handleIncrement`, `handleDecrement`, `WeightFormSchema`, and all other card logic remain unchanged. InlineEdit just provides another way to set `pendingWeight`.

**Why this works:** The InlineEdit component calls `onValueChange(roundedValue)` on blur, which maps to `setPendingWeight`. This is the exact same state setter that `handleIncrement` and `handleDecrement` use. The existing "Record Weight" button and `handleSubmit` flow work identically regardless of how `pendingWeight` was set.

```typescript
test('should allow direct keyboard entry for weight', async ({ page }) => {
  await page.goto('/');
  const weightCard = page.locator('#section-weight');
  await weightCard.scrollIntoViewIfNeeded();
  await expect(weightCard).toBeVisible();

  // Wait for card to initialize
  const recordBtn = weightCard.locator('button:has-text("Record Weight")');
  await expect(recordBtn).toBeVisible();

  // Find the hidden input via data-testid
  const weightInput = weightCard.getByTestId('weight-direct-input');

  // Click the weight display area to focus the hidden input
  // Use the label/display area click which triggers focus on the hidden input
  await weightInput.focus();

  // Clear and type a new value
  await weightInput.fill('71.35');

  // Blur to trigger rounding (tab away)
  await recordBtn.focus();

  // Wait for display to update with the rounded value
  // 71.35 is already aligned to 0.05 increments, so it stays 71.35
  await expect(weightCard.getByText('71.35')).toBeVisible();

  // Submit via existing Record Weight button
  await recordBtn.click();

  // Verify success toast
  await expect(page.getByText('Weight recorded', { exact: true })).toBeVisible();
});
```

**Notes:**
- `getByTestId('weight-direct-input')` targets the hidden input element directly (Playwright can interact with sr-only elements via fill/focus)
- `fill('71.35')` clears the input and types the value — this is the standard Playwright way to set input values
- After blur, the display should show `71.35` (already aligned to 0.05 increment)
- The test verifies the full flow: focus → type → blur (round) → submit → success toast
