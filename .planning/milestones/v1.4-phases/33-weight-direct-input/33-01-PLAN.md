Create `src/components/ui/inline-edit.tsx` with the following structure:

1. **"use client"** directive at top (component uses useState, event handlers)

2. **Props interface** (`InlineEditProps`):
   - `value: number | null` — current numeric value
   - `onValueChange: (value: number) => void` — callback on blur after rounding
   - `formatDisplay: (value: number | null) => string` — format function for the visible text (e.g., `v => v?.toFixed(2) ?? "--"`)
   - `roundOnBlur?: (value: number) => number` — optional rounding function applied on blur
   - `suffix?: string` — text rendered after the value (e.g., "kg")
   - `displayClassName?: string` — Tailwind classes for the value display span
   - `suffixClassName?: string` — Tailwind classes for the suffix span
   - `min?: number` — minimum valid value (default 0)
   - `max?: number` — maximum valid value (default 100000)
   - Extend `Omit<React.InputHTMLAttributes<HTMLInputElement>, 'value' | 'onChange' | 'onBlur' | 'min' | 'max'>` for pass-through props (type, inputMode, step, className, aria-label, data-testid, etc.)

3. **Component** (`React.forwardRef<HTMLInputElement, InlineEditProps>`):
   - Internal state: `isEditing: boolean` (false), `editValue: string` ("")
   - `handleFocus`: set `isEditing = true`, set `editValue` to `String(value ?? "")`, select all text in input
   - `handleBlur`: 
     - If `editValue` is empty or NaN: revert (set `isEditing = false`, do NOT call `onValueChange`)
     - Otherwise: parse float, clamp to [min, max], apply `roundOnBlur` if provided, call `onValueChange(result)`, set `isEditing = false`
   - `handleChange(e)`: set `editValue = e.target.value`
   - `handleKeyDown(e)`: if Enter, blur the input (`e.currentTarget.blur()`)

4. **Render structure** (per D-03, D-05, D-06):
   ```tsx
   <label className={cn("cursor-text inline-flex items-baseline", className)}>
     <span className={displayClassName}>
       {isEditing ? editValue || formatDisplay(null) : formatDisplay(value)}
     </span>
     {suffix && <span className={suffixClassName}>{suffix}</span>}
     <input
       ref={ref}
       type={type ?? "text"}
       inputMode={inputMode}
       step={step}
       className="sr-only"
       value={isEditing ? editValue : ""}
       onChange={handleChange}
       onFocus={handleFocus}
       onBlur={handleBlur}
       onKeyDown={handleKeyDown}
       aria-label={ariaLabel}
       data-testid={dataTestId}
       {...restProps}
     />
   </label>
   ```

5. **Display name**: `InlineEdit.displayName = "InlineEdit"`

6. **Export**: `export { InlineEdit }` and `export type { InlineEditProps }`

**Important implementation details:**
- The `sr-only` class (Tailwind) makes the input visually hidden but still focusable and accessible
- During editing, the display span shows the raw `editValue` string so the user sees exactly what they're typing
- The `formatDisplay(null)` fallback during editing when editValue is empty shows the placeholder format (e.g., "--")
- Pass through `data-testid` and `aria-label` from props to the hidden input for Playwright testability (per D-09)
- Do NOT add any borders, rings, or background changes on focus — the only visual cue is the display text changing as the user types (per D-05, UI-SPEC)
