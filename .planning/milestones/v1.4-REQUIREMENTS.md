# Requirements: Intake Tracker

**Defined:** 2026-04-05
**Core Value:** Accurate, queryable health data across intake, vitals, and medication adherence -- structured for cross-domain analysis and future AI querying

## v1.4 Requirements

Requirements for Post-Release Fixes milestone. Each maps to roadmap phases.

### CI/Release

- [ ] **REL-01**: User's release-please workflow can create PRs (fix GitHub Actions permissions in repo settings)

### Weight Tracking

- [ ] **WGT-01**: User can tap the weight value and type a number directly via keyboard
- [ ] **WGT-02**: User can configure weight increment step size in Settings (default 0.05, fix parseInt->parseFloat in settings-helpers)
- [ ] **WGT-03**: Weight rounding supports 0.05 precision (fix hardcoded *10/10 rounding to *100/100)

### Food/Sodium Card

- [ ] **FOOD-01**: Sodium amount displays in top-right of food card for consistency with other cards
- [ ] **FOOD-02**: Food description field is used as the primary entry title/label
- [ ] **FOOD-03**: Single merged history section at card bottom showing date, amount, and truncated name

### Presets

- [ ] **PRES-01**: AI auto-populates caffeine/alcohol content when user creates a coffee preset via substance-lookup
- [ ] **PRES-02**: User can delete presets from the preset grid (add missing delete UI)

## Future Requirements

Deferred to future release. Tracked but not in current roadmap.

### Presets

- **PRES-F01**: Debounced auto-trigger of AI lookup on preset name input (currently requires explicit button press)
- **PRES-F02**: Edit-mode toggle for preset grid (iOS home screen style jiggle/delete)

### Weight

- **WGT-F01**: Manual input dialog pattern for weight (alternative to inline tap-to-type)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Release-please PAT for CI on release PRs | GITHUB_TOKEN limitation; low impact since CI runs on source PRs |
| Composable entry grouping in merged history | Complex UX decision; individual entries sorted by timestamp is sufficient for v1.4 |
| Preset deletion from customization panel | Current scope is grid-level deletion only; panel is a settings context |
