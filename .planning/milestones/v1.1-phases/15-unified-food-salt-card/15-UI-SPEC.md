## Color

This phase uses **two domain-specific color tokens** for the two stacked sections. The card uses the `eating` theme as the primary card gradient (food is the dominant section) with the salt section using `salt` theme tokens for its interactive elements.

### Section-Specific Theming

| Section | Theme Source | Interactive Elements |
|---------|-------------|---------------------|
| Card wrapper | `CARD_THEMES.eating` | Card gradient, border, header icon |
| Food section | `CARD_THEMES.eating` | "I ate" button, AI sparkle icon, "Confirm All" button, record detail submit |
| Salt section | `CARD_THEMES.salt` | +/- buttons, Confirm Entry button, progress bar, over-limit warning |
| Preview cards | Neutral (`bg-muted/50 border`) | Edit fields use foreground text; remove button uses `text-muted-foreground hover:text-destructive` |

### Card Surface Color Contract

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-gradient-to-br from-orange-50 to-amber-50 dark:from-orange-950/40 dark:to-amber-950/40` | Card background gradient (eating theme) |
| Secondary (30%) | `bg-muted/50` | Preview card backgrounds, expandable detail containers, section divider background |
| Accent (10%) | `CARD_THEMES.eating.buttonBg` (`bg-orange-600 hover:bg-orange-700`) | "I ate" button, "Confirm All" button, AI sparkle icon active state |
| Destructive | `hsl(var(--destructive))` | Remove (X) button on preview cards hover, error toast, over-limit salt warning |

### Accent Reserved For (by section)

**Food section:**
- "I ate" quick-log button fill color
- AI sparkle icon tint when idle (`text-orange-600 dark:text-orange-400`)
- "Confirm All" button fill color in preview
- "Record with details" submit button in expanded details

**Salt section:**
- +/- button hover highlight (`hover:bg-amber-100`)
- Confirm Entry button fill color (`bg-amber-600`)
- Progress bar indicator fill
- Over-limit warning uses `text-red-600 dark:text-red-400` (not theme accent)

### Section Divider

A visual separator between food and salt sections: `border-t border-border/50` with `my-4` margin (16px above and below). This is a soft divider, not a hard boundary -- both sections remain within the same card.

## Layout Specifications

### Card Header

```
[icon-box 36px] [8px gap] ["FOOD + SALT" heading]       [last ate time]
```

- Icon box: 36px (p-2 + w-5 h-5 icon = existing pattern from LiquidsCard/IntakeCard)
- Header icon: `Utensils` from CARD_THEMES.eating (same icon as current EatingCard)
- Header label: "FOOD + SALT" -- `font-semibold text-lg uppercase tracking-wide`
- Right side: last eating timestamp in `text-xs text-muted-foreground` (same as current EatingCard pattern)
- Card gradient: `CARD_THEMES.eating.gradient` (from-orange-50 to-amber-50)

### Food Section Layout

```
[ I ate (h-11, full width, orange-600) ]              mb-2

[ Add details (ghost, chevron) ]                      mb-0 (collapsed)

-- when details expanded --                           p-3 bg-muted/50 border rounded-lg
  [What I ate (textarea, min-h-[80px]) ]              space-y-2
  [Weight in grams (input, number) ]                  space-y-2
  [When (datetime-local input) ]                      space-y-2
  [Record with details (button) ]                     mt-2

[ food description input_____ (sparkle) ]             mt-3, mb-0

-- when AI preview visible --
  ComposablePreview                                   mt-3
```

#### "I ate" Quick-Log Button

- Full-width `Button`, `h-11`, `bg-orange-600 hover:bg-orange-700`
- Icon: `Check` (w-4 h-4) left of label
- Label: "I ate"
- Loading state: `Loader2 animate-spin` replaces content
- Logs an eating record with no details (timestamp only), same as current EatingCard

#### "Add details" Expandable

- `Button variant="ghost" size="sm"` full-width with `justify-between`
- Left: "Add details" text in `text-muted-foreground`
- Right: `ChevronDown` (collapsed) / `ChevronUp` (expanded) icon, `w-4 h-4`
- Expanded content: `Collapsible` wrapper with `p-3 rounded-lg bg-muted/50 border space-y-3`
- Fields: Textarea for "What I ate" (optional), Input for grams (optional), Input for datetime-local "When"
- Submit button: "Record with details", full-width, `bg-orange-600`

#### AI Food Input

- Single row: `Input` component, full-width, `h-10`
- Placeholder: "Describe food for AI parsing..."
- Right side: inline button with `Sparkles` icon (16px), positioned inside the input using relative/absolute positioning (same pattern as Phase 14 PresetTab)
- Idle: icon in `text-muted-foreground`
- Loading: icon replaced with `Loader2` spinner (16px, `animate-spin`)
- Trigger: tap sparkle icon or press Enter key
- Success: preview populates below the input
- Error: toast with "AI parsing failed. Try again or add details manually."
- Disabled state: sparkle button disabled when input is empty

### Composable Preview Layout

```
Preview: "Two eggs with toast and butter"              text-sm font-medium mb-2

[Eating] Two eggs with toast and butter               Collapsible card
  [edit: description, grams]               [X]        inline fields when expanded

[Water] 85 ml                                         Collapsible card
  [edit: amount (ml)]                      [X]        inline fields when expanded

[Salt] 890 mg                                         Collapsible card
  [edit: amount (mg)]                      [X]        inline fields when expanded

AI reasoning: "Eggs ~75% water..."                    text-xs italic text-muted-foreground

[Try Again (outline)] [Confirm All (filled)]          grid grid-cols-2 gap-3, mt-3
```

#### Preview Container

- Wrapper: `space-y-2 mt-3`
- Original input text: `text-sm font-medium mb-2` showing what was typed

#### Record Mini-Cards

Each linked record is a mini-card with:

- Container: `p-3 rounded-lg bg-muted/50 border flex items-center justify-between`
- Left side: record type icon (14px, muted) + type label + amount summary
  - Eating: `Utensils` icon + "Eating" label + description text (truncated)
  - Water: `Droplets` icon + "Water" label + "{amount} ml"
  - Salt: `Sparkles` icon + "Salt" label + "{amount} mg"
- Right side: remove button (`X` icon, 16px, `text-muted-foreground hover:text-destructive`)
- Tap anywhere on the card (except X) expands inline edit fields below:
  - Eating: description `Input` + grams `Input` (optional)
  - Water: amount `Input type="number"` with "ml" suffix
  - Salt: amount `Input type="number"` with "mg" suffix
- Fields pre-filled from AI parse values
- Editing is inline within the card (no separate dialog). Changes update the preview state immediately.

#### Record Removal

- Tapping X on a record removes it from the preview (not yet persisted -- just removed from local state)
- At least one record must remain for Confirm All to be enabled
- If all records removed, show "No records to save. Try again or add details manually." in `text-sm text-muted-foreground`

#### AI Reasoning

- Below the record cards: `text-xs italic text-muted-foreground`
- Only shown if `reasoning` field is present in AI response
- Truncated to 2 lines with `line-clamp-2`

#### Preview Action Buttons

- Two buttons side by side: `grid grid-cols-2 gap-3 mt-3`
- "Try Again" button: `Button variant="outline"`, h-12, full-width. Clears preview, clears text input, returns to input state.
- "Confirm All" button: `Button variant="default"`, h-12, `bg-orange-600 hover:bg-orange-700`, full-width.
- "Confirm All" disabled when: `isSubmitting || previewRecords.length === 0`
- Loading state for Confirm All: text changes to "Saving..."

## Interaction States

### Quick "I ate" Button

| State | Visual |
|-------|--------|
| Default | Orange filled button with Check icon + "I ate" label |
| Loading | Loader2 spinner replaces button content |
| Success | Toast: "Logged" / "Eating event recorded". Button returns to default. |
| Error | Toast: "Error" / "Failed to record". Button returns to default. |

### "Add details" Expandable

| State | Visual |
|-------|--------|
| Collapsed | Ghost button with "Add details" + ChevronDown icon |
| Expanded | Ghost button with "Add details" + ChevronUp icon. Detail form visible below in bg-muted/50 container. |
| Submitting | Submit button in expanded area shows Loader2 spinner |
| AI preview visible | "Add details" section is hidden (mutually exclusive with AI preview) |

### AI Food Input

| State | Visual |
|-------|--------|
| Idle | Sparkles icon in `text-muted-foreground`, input enabled |
| Triggered | Input disabled, Sparkles replaced by `Loader2 animate-spin` |
| Success | Input re-enabled, sparkle icon returns, ComposablePreview appears below. Input retains search text. |
| Error | Input re-enabled, sparkle icon returns, destructive toast: "AI parsing failed. Try again or add details manually." |
| Empty input + tap AI | No-op. Sparkle button visually disabled when input is empty. |

### Composable Preview

| State | Visual |
|-------|--------|
| Preview shown | Record mini-cards visible with edit/remove capabilities |
| Record expanded | Tapped record shows inline edit fields. Other records remain collapsed. |
| Record removed | Card animates out (opacity-0 transition-opacity duration-200). Remaining cards reflow. |
| All removed | "No records to save. Try again or add details manually." message replaces cards. Confirm All disabled. |
| Editing a value | Input field in expanded card. Value updates preview state in real-time. |
| Confirm All | All records saved atomically. Toast: "Food logged" / "{count} linked records created". Preview clears. Input clears. |
| Confirm All loading | Button text "Saving...", disabled state. |
| Try Again | Preview clears. Input clears. Returns to input-ready state. |

### Salt Section States (preserved from IntakeCard(salt))

| State | Visual |
|-------|--------|
| Default | Pending amount = saltIncrement setting, +/- buttons enabled |
| At minimum | Minus button disabled (pendingAmount <= increment) |
| Would exceed limit | Pending amount text turns `text-orange-600 dark:text-orange-400` |
| Over limit | Progress bar turns `bg-red-500`, daily total turns `text-red-600 dark:text-red-400` |
| Submitting | Confirm button disabled, text "Recording..." |
| Center tap | Opens ManualInputDialog |

### Food + Salt Section Mutual Independence

- Food section and salt section operate independently. Logging salt does not affect the food section and vice versa.
- Exception: AI food parse that includes salt creates a salt intake record via composable entry. This salt amount appears in the salt section's daily total via `useIntake("salt")` reactivity (useLiveQuery).
- No explicit coordination needed -- Dexie's live queries handle cross-section reactivity automatically.

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Card landmark | Card has no explicit landmark -- standard `div` with Card styling (existing pattern) |
| Quick log button | Accessible name from button text content "I ate" |
| Add details toggle | `aria-expanded` via Collapsible primitives, `aria-controls` on CollapsibleTrigger |
| AI input | `aria-label="Describe food for AI nutritional parsing"` on the input |
| AI button | `aria-label="Parse food with AI"` on the sparkle icon button |
| Preview record cards | Each card has a `button` role for expand/collapse with `aria-expanded` state |
| Remove button | `aria-label="Remove {type} record"` on each X button (e.g. "Remove water record") |
| Salt +/- buttons | Accessible names from icon + context ("Decrease amount" / "Increase amount") |
| Loading states | Buttons use `disabled` attribute; spinner has `aria-hidden="true"` |
| Form fields | Associated `Label` components with `htmlFor` in both detail form and preview edit |
| Color contrast | All text meets WCAG 2.1 AA (existing theme tokens verified in v1.0) |
| Reduced motion | Card transitions respect `prefers-reduced-motion` via existing CSS media query |
| Focus management | After AI parse returns, focus moves to the first preview card. After Confirm All, focus returns to the AI input. |
