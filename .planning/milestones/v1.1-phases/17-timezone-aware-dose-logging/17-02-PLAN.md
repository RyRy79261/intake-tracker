1. Create `src/hooks/use-timezone-detection.ts`:

The hook manages timezone change detection and dialog state. Key implementation details:

**Module-level session flag (per D-07):**
```typescript
let _dismissedThisSession = false;
```
This resets naturally on page reload/app restart.

**Hook signature and state:**
```typescript
export interface TimezoneChangeState {
  dialogOpen: boolean;
  oldTimezone: string;
  newTimezone: string;
  isRecalculating: boolean;
  handleConfirm: () => Promise<void>;
  handleDismiss: () => void;
}

export function useTimezoneDetection(): TimezoneChangeState
```

**Detection logic (checkTimezoneChange function inside hook):**
- If `_dismissedThisSession` is true, return early
- Call `clearTimezoneCache()` to bust the cache
- Call `getDeviceTimezone()` to get current IANA timezone
- Query `db.phaseSchedules.toArray()`, filter to `enabled === true`, get unique `anchorTimezone` values
- If no active schedules exist, return early (nothing to compare)
- Compare device timezone against stored anchor timezones
- If any schedule has a different IANA name: set `dialogOpen=true`, store `oldTimezone` (from schedule) and `newTimezone` (device)
- Per D-06: If the IANA name is the SAME but offset differs (DST transition within same zone), this cannot happen -- same IANA name means same zone. DST transitions are invisible because `recalculateScheduleTimezones` uses the IANA name for offset calculation, which automatically accounts for DST. So no explicit DST handling is needed in the detection logic.

**IMPORTANT: The detection must import `db` from `@/lib/db` directly.** This is a hook file, not a component, so the ESLint service-boundary rule (`no-restricted-imports`) does not apply to hooks (hooks ARE the service boundary layer -- see `src/hooks/use-medication-queries.ts` which imports from services).

**handleConfirm callback:**
- Set `isRecalculating = true`
- Call `await recalculateScheduleTimezones(newTimezone)` (imported from `@/lib/timezone-recalculation-service`)
- Show success toast: `toast({ title: "Schedules adjusted to {cityName}" })` where cityName is extracted from the IANA name using `newTimezone.split("/").pop()?.replace(/_/g, " ") ?? newTimezone`
- Set `dialogOpen = false`, `isRecalculating = false`
- On error: show error toast with title "Schedule adjustment failed" and description "Your dose times have not changed. Try reopening the app." Per UI-SPEC error state.

**handleDismiss callback:**
- Set `_dismissedThisSession = true`
- Set `dialogOpen = false`

**Visibility change listener (per D-04):**
```typescript
useEffect(() => {
  const handleVisibilityChange = () => {
    if (document.visibilityState === "visible") {
      checkTimezoneChange();
    }
  };

  // Check on mount (app open)
  checkTimezoneChange();

  // Check on resume from background
  document.addEventListener("visibilitychange", handleVisibilityChange);
  return () => {
    document.removeEventListener("visibilitychange", handleVisibilityChange);
  };
}, []);
```

**City name extraction helper (for display in dialog and toasts):**
```typescript
function formatTimezoneCityName(iana: string): string {
  return iana.split("/").pop()?.replace(/_/g, " ") ?? iana;
}
```

import { useTimezoneDetection } from "@/hooks/use-timezone-detection";
import { TimezoneChangeDialog } from "@/components/medications/timezone-change-dialog";

function TimezoneGuard({ children }: { children: React.ReactNode }) {
  const {
    dialogOpen,
    oldTimezone,
    newTimezone,
    isRecalculating,
    handleConfirm,
    handleDismiss,
  } = useTimezoneDetection();

        open={dialogOpen}
        oldTimezone={oldTimezone}
        newTimezone={newTimezone}
        isRecalculating={isRecalculating}
        onConfirm={handleConfirm}
        onDismiss={handleDismiss}
      />
    </>
  );
}
