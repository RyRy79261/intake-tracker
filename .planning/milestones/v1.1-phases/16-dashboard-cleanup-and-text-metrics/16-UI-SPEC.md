## Color

This phase uses the existing **CARD_THEMES** domain-specific color tokens for each metric row in the text metrics display. No new color tokens are introduced.

### Text Metrics Color Mapping

Each metric row uses its domain theme color for the progress bar fill and the value text:

| Metric | Theme Source | Progress Fill | Value Color |
|--------|-------------|---------------|-------------|
| Water | `CARD_THEMES.water` | `from-sky-400 to-cyan-500` | `text-sky-700 dark:text-sky-300` |
| Salt | `CARD_THEMES.salt` | `from-amber-400 to-orange-500` | `text-amber-700 dark:text-amber-300` |
| Caffeine | `CARD_THEMES.caffeine` | `from-yellow-400 to-amber-500` | `text-yellow-700 dark:text-yellow-300` |
| Alcohol | `CARD_THEMES.alcohol` | `from-fuchsia-400 to-pink-500` | `text-fuchsia-700 dark:text-fuchsia-300` |

### Surface Color Contract

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` | Page background, text metrics section background |
| Secondary (30%) | `bg-muted/50` | Text metrics container surface, weekly summary container, collapsible detail backgrounds, Liquid Presets manager list items |
| Accent (10%) | Per-metric domain color (see table above) | Progress bar fills, metric value text, over-limit warning, heart rate icon |
| Destructive | `hsl(var(--destructive))` | Over-limit progress bar (`bg-red-500`), over-limit value text (`text-red-600 dark:text-red-400`) |

### Accent Reserved For

- Progress bar indicator fills (one per metric, each in its domain color)
- Metric value text when within limit (domain color)
- Over-limit value text (switches to `text-red-600 dark:text-red-400`)
- Over-limit progress bar (switches to `bg-red-500`)
- Heart rate icon tint in BP card (`text-rose-600 dark:text-rose-400`)
- "Save" / action buttons in Liquid Presets manager (using parent dialog accent)

## Layout Specifications

### Visual Focal Point

**Primary visual anchor: TextMetrics section with per-domain colored progress bars.** The TextMetrics container is the first content block below InsightBadge and above all domain cards. Its domain-colored progress bars (water sky-to-cyan, salt amber-to-orange) against the neutral `bg-muted/50` surface create the strongest color contrast on the page, drawing the eye immediately on load. All other cards sit below in a secondary reading flow.

### Dashboard Page Order (D-09)

```
[AppHeader]

[InsightBadge]                                    mb-4 (existing)

[TextMetrics]                                     mb-6

[LiquidsCard]                                     space-y-4
[FoodSaltCard]

[BloodPressureCard]                               space-y-4
[WeightCard]

[UrinationCard]                                   space-y-4
[DefecationCard]

[Footer]
```

- InsightBadge remains in its existing position above all content
- TextMetrics replaces the HistoricalGraph slot
- Card groupings preserved: intake cards (Liquids, Food+Salt), health cards (BP, Weight), bodily function cards (Urination, Defecation)
- BP card moves above Weight card (current order is Weight then BP -- reversed per D-09)

### TextMetrics Layout (D-01, D-02, D-03)

```
Today                                             text-xs font-medium uppercase tracking-wide text-muted-foreground mb-3

Water     ===========================  1,200 / 2,000 ml     row, gap-3
Salt      ===========                    890 / 1,500 mg     row, gap-3
Caffeine                                      420 mg        row, gap-3
Alcohol                                 0.0 std drinks      row, gap-3

This Week (Mon-Sun)                               mt-4, text-xs font-medium uppercase tracking-wide text-muted-foreground mb-2

Mon  Tue  Wed  Thu  Fri  Sat  Sun                 grid grid-cols-7
850  920 1200  ---  ---  ---  ---  ml (water)      text-xs tabular-nums text-center
700  800  890  ---  ---  ---  ---  mg (salt)        text-xs tabular-nums text-center
```

#### TextMetrics Container

- Wrapper: `rounded-lg bg-muted/50 border p-4` (16px padding)
- No card header icon or gradient -- this is a neutral summary section, not a domain card
- Full width within the `max-w-lg` page container

#### Today's Metrics Rows

Each metric row is a horizontal layout:

```
[icon 16px] [label] [progress bar (flex-1)] [value / limit]
```

- Icon: domain icon from CARD_THEMES at 16px (`w-4 h-4`), colored per domain (`CARD_THEMES.{domain}.iconColor`)
- Label: `text-sm` (14px), `text-foreground`, `w-16` fixed width for alignment
- Progress bar (water and salt only): `Progress` component, `h-2` (thin), flex-1, `indicatorClassName` from `CARD_THEMES.{domain}.progressGradient`
- Value: `text-sm font-semibold tabular-nums` in domain `latestValueColor`
- Limit suffix: `text-xs text-muted-foreground` for "/ 2,000 ml"
- Over-limit: value text turns `text-red-600 dark:text-red-400`, progress bar indicator turns `bg-red-500`
- Row spacing: `space-y-2` between rows (8px)

**Water row:** icon=Droplets, progress bar visible, value="1,200 / 2,000 ml"
**Salt row:** icon=Sparkles, progress bar visible, value="890 / 1,500 mg"
**Caffeine row:** icon=Coffee, NO progress bar (no limit), value="420 mg"
**Alcohol row:** icon=Wine, NO progress bar (no limit), value="0.0 std drinks"

Caffeine and alcohol rows omit the progress bar. The value is right-aligned to match the limit-based rows. Layout:

```
[icon 16px] [label] [spacer flex-1] [value]
```

#### Weekly Summary

- Section heading: "This Week (Mon-Sun)" in `text-xs font-medium uppercase tracking-wide text-muted-foreground`
- Grid: `grid grid-cols-7 gap-1`
- Day headers: `text-xs text-muted-foreground text-center font-medium` -- "M", "T", "W", "T", "F", "S", "S" (single-letter abbreviations)
- Water row values: `text-xs tabular-nums text-center` in `CARD_THEMES.water.latestValueColor`
- Salt row values: `text-xs tabular-nums text-center` in `CARD_THEMES.salt.latestValueColor`
- Future days: show "---" in `text-muted-foreground/50`
- Today's column: `font-semibold` to distinguish from past days
- Row labels: `text-xs text-muted-foreground` left of the grid row -- "Water" and "Salt" in a fixed-width column before the 7-day grid
- Over-limit days: value text turns `text-red-600 dark:text-red-400`
- Layout for weekly grid:

```
          M    T    W    T    F    S    S
Water   850  920 1200  ---  ---  ---  ---
Salt    700  800  890  ---  ---  ---  ---
```

Implemented as:

```
grid grid-cols-[auto_repeat(7,1fr)] gap-x-1 gap-y-1
```

The `auto` first column holds the row label. Each of the 7 `1fr` columns holds a day value.

## Interaction States

### TextMetrics

| State | Visual |
|-------|--------|
| Loading | Skeleton placeholders: 4 rows of `h-2 w-full rounded bg-muted animate-pulse` |
| No data today | All values show "0" with empty progress bars (0%). Not an error state -- just a fresh day. |
| Over limit (water) | Water value turns `text-red-600`, progress bar indicator turns `bg-red-500` |
| Over limit (salt) | Salt value turns `text-red-600`, progress bar indicator turns `bg-red-500` |
| No caffeine logged | Caffeine row shows "0 mg" in muted foreground |
| No alcohol logged | Alcohol row shows "0.0 std drinks" in muted foreground |
| Weekly future days | Show "---" in `text-muted-foreground/50` |
| Weekly past days with no data | Show "0" in `text-muted-foreground/50` |

### BP Card Heart Rate (promoted)

| State | Visual |
|-------|--------|
| Default | Empty input with placeholder "72", BPM suffix visible |
| Value entered | Number displayed in input, BPM suffix unchanged |
| Validation error | `text-sm text-destructive` below input: "Heart rate must be between 0 and 250" |
| Submit without HR | Allowed -- heart rate remains optional. BP recorded with null heart rate. |
| Submit with HR | BP recorded with heart rate value. All fields clear on success. |

### Customization Panel -- Liquid Presets

| State | Visual |
|-------|--------|
| Empty list | "No custom presets. Default presets are always available." in `text-sm text-muted-foreground` |
| Default preset | Name + badge "Default", no delete button, edit button available |
| Custom preset | Name + edit + delete buttons |
| Delete custom | Confirmation: "Delete {name}?" with "Keep Preset" and "Delete Preset" buttons. "Delete Preset" button uses `variant="destructive"`. |
| Edit preset | Inline form or dialog with fields pre-filled. Save button: "Save Changes". Cancel dismisses without saving. |
| Add preset | Same form as edit but empty. Save button: "Add Preset". |

### Multi-Substance Preset Logging (D-11)

| State | Visual |
|-------|--------|
| Select multi-substance preset | Volume and all per-100ml fields populate. Calculated display shows all substances: "420 mg caffeine, 1.2 std drinks" |

| Success | Toast: "Logged {name}" with description listing created records |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| TextMetrics landmark | `section` element with `aria-label="Daily intake summary"` |
| Metric icons | `aria-hidden="true"` on all decorative icons in metric rows |
| Progress bars | Each `Progress` has `aria-label="{Type} intake progress"` and `aria-valuenow`, `aria-valuemin=0`, `aria-valuemax={limit}` (provided by Radix Progress) |
| Weekly summary | Grid uses `role="grid"` with `role="columnheader"` on day headers and `role="row"` on data rows, or plain `div` grid since it is presentational (screen reader sees the text content) |
| BP heart rate field | `Label htmlFor="heartrate"` with descriptive text; `aria-describedby` links to validation error if present |
| Preset list | Each preset item is a plain `div` (not interactive); edit/delete buttons have `aria-label="Edit {name}"` / `aria-label="Delete {name}"` |
| Delete confirmation | Focus trapped in confirmation dialog; Escape dismisses |
| Color contrast | All text meets WCAG 2.1 AA. Over-limit red (`text-red-600` on white bg = 4.8:1 ratio; `text-red-400` on dark bg = 4.6:1 ratio) |
| Reduced motion | Progress bar transition respects `prefers-reduced-motion` via existing `transition-all duration-300` which Radix honors |
