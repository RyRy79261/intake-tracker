Create `.github/workflows/ci.yml` with the following exact structure. This implements D-01 (single workflow, parallel jobs), D-02 (PR to main trigger), D-03 (two separate TZ jobs), D-04 (pnpm cache), D-05 (bundle security in build job), D-07 (status checks only), and D-08 (ci-pass gate).

```yaml
name: CI

on:
  pull_request:
    branches: [main]

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v5
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'pnpm'
      - run: pnpm install --frozen-lockfile
      - run: pnpm lint

  typecheck:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v5
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'pnpm'
      - run: pnpm install --frozen-lockfile
      - run: pnpm typecheck

  test-tz-sa:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v5
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'pnpm'
      - run: pnpm install --frozen-lockfile
      - name: Run tests (Africa/Johannesburg)
        run: pnpm test:tz:sa

  test-tz-de:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v5
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'pnpm'
      - run: pnpm install --frozen-lockfile
      - name: Run tests (Europe/Berlin)
        run: pnpm test:tz:de

  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v5
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'pnpm'
      - run: pnpm install --frozen-lockfile
      - name: Build production bundle
        run: pnpm build
      - name: Bundle security scan
        run: pnpm exec vitest run src/__tests__/bundle-security.test.ts

  ci-pass:
    if: always()
    needs: [lint, typecheck, test-tz-sa, test-tz-de, build]
    runs-on: ubuntu-latest
    steps:
      - name: Check all jobs passed
        run: |
          if [[ "${{ needs.lint.result }}" != "success" ||
                "${{ needs.typecheck.result }}" != "success" ||
                "${{ needs.test-tz-sa.result }}" != "success" ||
                "${{ needs.test-tz-de.result }}" != "success" ||
                "${{ needs.build.result }}" != "success" ]]; then
            echo "::error::One or more CI jobs failed or were cancelled"
            exit 1
          fi
```

Key implementation details:
- **Node 20** (not 18 from version-bump.yml): Node 18 is EOL since April 2025. Node 20 is active LTS. Per Claude's Discretion.
- **pnpm/action-setup@v5** without `version:` input: reads `packageManager: pnpm@10.30.2` from package.json automatically. Per D-04.
- **actions/setup-node@v4 with `cache: 'pnpm'`**: handles pnpm store caching via pnpm-lock.yaml hash. Per D-04. Do NOT also use `pnpm/action-setup` cache or `actions/cache` (avoid dual-caching anti-pattern from RESEARCH.md).
- **`--frozen-lockfile`**: prevents lockfile updates in CI.
- **`if: always()`** on ci-pass: ESSENTIAL. Without it, if any upstream job fails, the gate job is skipped, and GitHub treats "skipped" as passing for required status checks (per RESEARCH.md Pitfall 2).
- **Explicit per-job result checks**: Do NOT use `contains(needs.*.result, 'failure')` -- there is an unfixed GitHub Actions runner bug (actions/runner#1540) where the wildcard includes transitive dependencies (per RESEARCH.md anti-patterns).
- **No PR annotations or comment summaries**: Each job reports independently via GitHub status checks (per D-07).
