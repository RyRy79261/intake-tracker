```typescript
// After:
```

**Pattern B: "Property 'data' does not exist on ServiceResult" (TS2339) -- 21 errors**

File: round-trip.test.ts (lines 204, 248-251, 274-280, 300, 312, 338-342, 383-384).

The `ServiceResult<T>` type is a discriminated union: `{ success: true; data: T } | { success: false; error: string }`. TypeScript requires narrowing before accessing `.data`. Fix by adding a narrowing guard before the `.data` access block:

```typescript
// Before:
const result = await importBackup(file, "merge");
expect(result.data?.success).toBe(true);
expect(result.data?.prescriptionsImported).toBe(1);

// After:
const result = await importBackup(file, "merge");
expect(result.success).toBe(true);
if (!result.success) throw new Error("Expected success");
expect(result.data.prescriptionsImported).toBe(1);
```

Apply this pattern to every `result.data` access in round-trip.test.ts. For each test function:
- Replace `expect(result.data?.success).toBe(true)` with `expect(result.success).toBe(true)`
- Add `if (!result.success) throw new Error("Expected success");` right after
- Change all subsequent `result.data?.field` to `result.data.field` (no optional chaining needed after narrowing)
- For `importResult.data?.conflicts`, `importResult.data?.resolved` etc., same pattern

**Pattern C: exactOptionalPropertyTypes (TS2375) -- 1 error**

File: db-fixtures.ts line 318. The return type `{ eatingId?: string; substanceId?: string }` means those properties must be either `string` or absent (not `undefined`). The variables `eatingId` and `substanceId` are `string | undefined`. Fix by conditionally spreading:

```typescript
// Before (line 318):
return { groupId, eatingId, intakeIds, substanceId };

// After:
return {
  groupId,
  intakeIds,
  ...(eatingId !== undefined && { eatingId }),
  ...(substanceId !== undefined && { substanceId }),
};
```

**Pattern D: Missing required properties (TS2345) -- 1 error**

File: medication-schedule-service.test.ts line 131. The `addSchedule` function expects `Omit<PhaseSchedule, "id" | "createdAt" | "updatedAt" | "deletedAt" | "deviceId" | "enabled">`, which includes `scheduleTimeUTC` and `anchorTimezone`. The function actually computes these internally from `time`, but the type requires them. Fix by adding dummy values that get overwritten:

```typescript
// Before:
const result = await addSchedule({

  time: "09:00",
  dosage: 25,
  daysOfWeek: [1, 3, 5],
  unit: "mg",
  scheduleTimeUTC: 0, // computed by addSchedule from time + timezone
  anchorTimezone: "UTC", // computed by addSchedule from device timezone
});
```
