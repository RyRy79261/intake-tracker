Insert a new job named `supply-chain` AFTER the `e2e` job and BEFORE the `ci-pass` job. It follows the identical setup pattern as all other jobs (checkout, pnpm/action-setup, setup-node with Node 20 + pnpm cache, frozen-lockfile install). Per D-04, this is a dedicated job for clear failure attribution.

The job has two steps after setup:

**Step "Verify supply chain config" (per D-06, corrected to grep pnpm-workspace.yaml):**
```yaml
- name: Verify supply chain config
  run: |
    echo "Checking pnpm-workspace.yaml for required security settings..."
    missing=0
    for setting in minimumReleaseAge trustPolicy blockExoticSubdeps; do
      if ! grep -q "^${setting}:" pnpm-workspace.yaml; then
        echo "::error::Missing required security setting: ${setting} in pnpm-workspace.yaml"
        missing=1
      fi
    done
    if [ "$missing" -eq 1 ]; then
      echo "Add the missing settings to pnpm-workspace.yaml. See Phase 23 docs."
      exit 1
    fi
    echo "All supply chain settings present"
```

**Step "Audit dependencies" (per D-01, D-05):**
```yaml
- name: Audit dependencies
  run: pnpm audit --audit-level high
```

No `--prod` flag (per D-05: scans all deps including dev). The `--audit-level high` flag means exit code 1 only on high/critical findings; moderate/low are reported but don't fail (per D-01).
