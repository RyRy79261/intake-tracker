```yaml
  changes:
    runs-on: ubuntu-latest
    permissions:
      pull-requests: read
    outputs:
      src: ${{ steps.filter.outputs.src }}
      bench: ${{ steps.filter.outputs.bench }}
    steps:
      - uses: actions/checkout@v4

```

**4. Add `coverage` job with base-branch comparison** (per D-06, D-07, D-08, D-09):

Add after the `e2e` job. This is gated by the `src` paths-filter. Per D-07, the job runs coverage on BOTH the base branch and the PR branch, then passes the base-branch output to the report action via `json-summary-compare-path` so the PR comment shows a delta (per D-08), not absolute numbers. Coverage is informational only (per D-09). See Research Pitfall 5 and Open Question 1 -- approach (a): single job, sequential base-then-PR.

```yaml
  coverage:
    needs: [changes]
    if: needs.changes.outputs.src == 'true'
    runs-on: ubuntu-latest
    permissions:
      pull-requests: write
    steps:
      - uses: actions/checkout@v4
        with:
          ref: ${{ github.base_ref }}
      - uses: pnpm/action-setup@v5
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'pnpm'
      - run: pnpm install --frozen-lockfile
      - name: Run coverage (base branch)
        run: pnpm test:coverage
      - name: Save base coverage
        run: mkdir -p coverage-base && cp coverage/coverage-summary.json coverage-base/coverage-summary.json
      - uses: actions/checkout@v4
        with:
          clean: false
      - run: pnpm install --frozen-lockfile
      - name: Run coverage (PR branch)
        run: pnpm test:coverage
      - name: Report coverage
        if: always()
        uses: davelosert/vitest-coverage-report-action@v2
        with:
          json-summary-compare-path: coverage-base/coverage-summary.json
          file-coverage-mode: none
```

**How this works (per D-07, D-08):**
1. Checkout the **base branch** (`github.base_ref` -- the PR target, e.g. `main`)
2. Install deps and run `pnpm test:coverage` -- produces `coverage/coverage-summary.json` for the base
3. Copy that file to `coverage-base/coverage-summary.json` (safe location that won't be overwritten)
4. Checkout the **PR branch** (default ref, `clean: false` preserves `coverage-base/`)
5. Reinstall deps (lockfile may differ) and run `pnpm test:coverage` again -- produces PR coverage
6. `vitest-coverage-report-action` reads PR coverage from `coverage/coverage-summary.json` and base coverage from `coverage-base/coverage-summary.json`, posts a delta-only comment

The `file-coverage-mode: none` per D-08 suppresses file-by-file breakdown. The `json-summary-compare-path` input enables delta display (e.g., "+0.5% lines") instead of absolute numbers.

**5. Add `benchmark` job** (per D-13, D-14, D-15, D-16, D-17):

Add after the `coverage` job. Gated by the `bench` paths-filter (separate from `src`). Benchmarks are informational only.

```yaml
  benchmark:
    needs: [changes]
    if: needs.changes.outputs.bench == 'true'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v5
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'pnpm'
      - run: pnpm install --frozen-lockfile
      - name: Run benchmarks
        run: pnpm bench --run --compare benchmarks/results.json
```

Note: `pnpm bench --run` (not `bench:ci`) because we do NOT want to overwrite the committed baselines in CI. The `--compare` flag compares current results against the committed baselines and prints the comparison to stdout (CI logs per D-16).

**6. Update `ci-pass` gate** (per D-18, D-19):

Replace the entire `ci-pass` job with skip-aware logic. The `needs` array must include ALL jobs (including the new ones). Unconditional jobs must be `success`. Gated jobs accept `success` OR `skipped`.

```yaml
  ci-pass:
    if: always()
    needs: [changes, lint, typecheck, test-tz-sa, test-tz-de, build, data-integrity, e2e, supply-chain, coverage, benchmark]
    runs-on: ubuntu-latest
    steps:
      - name: Check all jobs passed
        run: |
          echo "=== Unconditional jobs ==="
          echo "changes: ${{ needs.changes.result }}"
          echo "lint: ${{ needs.lint.result }}"
          echo "typecheck: ${{ needs.typecheck.result }}"
          echo "build: ${{ needs.build.result }}"
          echo "data-integrity: ${{ needs.data-integrity.result }}"
          echo "supply-chain: ${{ needs.supply-chain.result }}"

          # Unconditional jobs: must succeed
          if [[ "${{ needs.changes.result }}" != "success" ||
                "${{ needs.lint.result }}" != "success" ||
                "${{ needs.typecheck.result }}" != "success" ||
                "${{ needs.build.result }}" != "success" ||
                "${{ needs.data-integrity.result }}" != "success" ||
                "${{ needs.supply-chain.result }}" != "success" ]]; then
            echo "::error::Unconditional job failed"
            exit 1
          fi

          echo "=== Gated jobs (success or skipped) ==="
          echo "test-tz-sa: ${{ needs.test-tz-sa.result }}"
          echo "test-tz-de: ${{ needs.test-tz-de.result }}"
          echo "e2e: ${{ needs.e2e.result }}"
          echo "coverage: ${{ needs.coverage.result }}"
          echo "benchmark: ${{ needs.benchmark.result }}"

          # Gated jobs: success OR skipped
          for result in \
            "${{ needs.test-tz-sa.result }}" \
            "${{ needs.test-tz-de.result }}" \
            "${{ needs.e2e.result }}" \
            "${{ needs.coverage.result }}" \
            "${{ needs.benchmark.result }}"; do
            if [[ "$result" != "success" && "$result" != "skipped" ]]; then
              echo "::error::Gated job failed (not success or skipped): $result"
              exit 1
            fi
          done

          echo "All jobs passed"
```
