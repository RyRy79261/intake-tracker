```markdown
### Pencil Design System

**Design-first workflow:** Each phase that changes UI starts with Pencil design updates, reviewed before coding.

- All designs live in `design/app.pen` (single file, version-controlled)
- Design tokens in Pencil mirror CSS custom properties in `globals.css`
- Domain colors: water (sky), salt (amber), weight (emerald), bp (rose), eating (orange), urination (violet), defecation (stone), caffeine (yellow), alcohol (fuchsia), medication (teal)
- Pencil requires running AppImage: `DISPLAY=:0 ~/Applications/squashfs-root/pencil --no-sandbox &`

**Naming conventions:**
- Screens: `Screen / {PageName}` (e.g., `Screen / Dashboard`)
- Screen states: `Screen / {PageName} {State}` (e.g., `Screen / Dashboard Empty`)
- Components: `{Category} / {Name}` (e.g., `Domain / Intake Card Water`)
- Overlays: `Overlay / {Name}` (e.g., `Overlay / Edit Intake`)
- Flows: `Flow / {Name} Step {N}` (e.g., `Flow / Add Prescription Step 1`)
- Chrome: `Chrome / {Name}` (e.g., `Chrome / App Header`)

**Design token flow:** `globals.css (source of truth) -> tailwind.config.ts (consumption) -> Pencil variables (mirror)`

**Update process for UI-changing phases:**
1. Update Pencil design for the UI change in `design/app.pen`
2. Verify with Pencil MCP `get_screenshot`
3. Implement in code to match design
4. Commit both `.pen` and code changes together

**Key conventions:**
- All colors use `$variable` references in Pencil (never hardcoded hex)
- Font: Outfit (not Inter)
- Screen width: 393px (iPhone 14 Pro logical width, matches mobile-first max-w-lg)
- Components marked `reusable: true` and instantiated via `ref` in screen designs
- Light/dark themes via Pencil theme axis (`mode: ["light", "dark"]`)
```

Do NOT modify any other section of CLAUDE.md. Only add the new section.
