2. Create `src/lib/analytics-types.ts` with all shared types:
   - `Domain` union type: 'water' | 'salt' | 'weight' | 'bp' | 'eating' | 'urination' | 'defecation' | 'caffeine' | 'alcohol' | 'medication'
   - `TimeScope` union: '24h' | '7d' | '30d' | '90d' | 'all'
   - `TimeRange` interface: `{ start: number; end: number }` (Unix ms timestamps)
   - `DataPoint`: `{ timestamp: number; value: number; label?: string }`
   - `AnalyticsResult<T>`: `{ value: T; unit: string; period: TimeRange; dataPoints: DataPoint[]; meta?: Record<string, unknown> }` -- plain serializable, no class instances
   - `FluidBalanceDay`: `{ date: string; intakeMl: number; urinationCount: number; urinationEstimatedMl: number; balance: number; target: number }`
   - `FluidBalanceResult`: `{ daily: FluidBalanceDay[]; intraday: DataPoint[]; avgBalance: number; daysAboveTarget: number; daysTotal: number }`
   - `AdherenceResult`: `{ rate: number; taken: number; total: number; daily: Array<{ date: string; rate: number; taken: number; total: number }> }`
   - `BPTrendResult`: `{ readings: Array<{ timestamp: number; systolic: number; diastolic: number; heartRate?: number; position: string }>; trend: { systolic: TrendDirection; diastolic: TrendDirection }; avg: { systolic: number; diastolic: number } }`
   - `TrendDirection`: `{ slope: number; direction: 'rising' | 'falling' | 'stable'; confidence: number }`
   - `WeightTrendResult`: `{ readings: DataPoint[]; trend: TrendDirection; avg: number; min: number; max: number }`
   - `CorrelationResult`: `{ coefficient: number; strength: 'strong' | 'moderate' | 'weak' | 'none'; seriesA: DataPoint[]; seriesB: DataPoint[]; lagDays: number }`
   - `InsightType` union: 'adherence_drop' | 'bp_trend' | 'weight_trend' | 'fluid_deficit' | 'correlation_alert' | 'anomaly'
   - `Insight`: `{ id: string; type: InsightType; title: string; description: string; severity: 'info' | 'warning' | 'alert'; value: number; threshold: number; timestamp: number }`
   - `TitrationReport`: `{ prescriptionId: string; prescriptionName: string; phases: Array<{ phaseId: string; phaseName: string; type: PhaseType; startDate: number; endDate?: number; adherence: AdherenceResult; bpDuring: BPTrendResult; weightDuring: WeightTrendResult }> }`
   - `URINATION_ESTIMATE_ML` constant map: `{ small: 150, medium: 300, large: 500 }` (for fluid balance proxy)
   - `DEFAULT_SALT_WEIGHT_LAG_DAYS` constant: `2` (per research recommendation)

3. Create `src/lib/analytics-stats.ts` wrapping simple-statistics:
   - `movingAverage(data: number[], windowSize: number): (number | null)[]` -- sliding window, returns null for insufficient data points
   - `trend(points: DataPoint[]): TrendDirection` -- uses `linearRegression` + `linearRegressionLine` from simple-statistics; direction = slope > 0.01 ? 'rising' : slope < -0.01 ? 'falling' : 'stable'; confidence from R-squared
   - `correlateTimeSeries(seriesA: DataPoint[], seriesB: DataPoint[], lagDays?: number): CorrelationResult` -- aligns by day, applies lag if specified, uses `sampleCorrelation`; strength thresholds: |r| > 0.7 strong, > 0.4 moderate, > 0.2 weak, else none
   - `detectAnomalies(points: DataPoint[], zThreshold?: number): DataPoint[]` -- uses `zScore` and `mean`/`standardDeviation`; default threshold 2.0; returns points exceeding threshold
   - `computeRegression(points: DataPoint[]): { slope: number; intercept: number; predict: (x: number) => number }` -- wraps `linearRegression` + `linearRegressionLine`
   - All functions handle edge cases: empty arrays return neutral results, single-element arrays return 'stable' trend with 0 confidence

Import from simple-statistics: `{ linearRegression, linearRegressionLine, sampleStandardDeviation, sampleCorrelation, zScore, mean }`.

4. Create `src/lib/analytics-stats.test.ts` with unit tests:
   - Test movingAverage with window=3 on [1,2,3,4,5] returns [null, null, 2, 3, 4]
   - Test movingAverage with empty array returns []
   - Test trend with rising data points returns direction='rising'
   - Test trend with stable data returns direction='stable'
   - Test trend with empty array returns 'stable' with confidence 0
   - Test correlateTimeSeries with perfectly correlated series returns coefficient ~1.0, strength='strong'
   - Test correlateTimeSeries with empty arrays returns coefficient=0, strength='none'
   - Test detectAnomalies flags points with z-score > 2.0
   - Test detectAnomalies with empty array returns []
   - Test computeRegression returns correct slope/intercept for linear data

<task type="auto">
  <name>Task 2: Create core analytics service with two-layer query architecture + tests</name>
  <files>src/lib/analytics-service.ts, src/lib/analytics-service.test.ts</files>
  <action>
Create `src/lib/analytics-service.ts` implementing the two-layer architecture:

**Layer 1 -- Building Blocks (exported for custom composition):**
- `getRecordsByDomain(domain: Domain, range: TimeRange): Promise<DataPoint[]>` -- dispatches to the correct domain service's `getXByDateRange(start, end)` and normalizes to DataPoint[]. For water: amount in ml. For salt: amount in mg. For weight: weight in kg. For bp: systolic as value. For urination: estimated ml using URINATION_ESTIMATE_ML map. For eating: 1 per event. For caffeine/alcohol: query substanceRecords (graceful if table doesn't exist yet -- return []).
- `groupByDay(points: DataPoint[]): Map<string, DataPoint[]>` -- groups DataPoint[] by calendar date (midnight-midnight boundaries per context decision, NOT day-start-hour). Use `startOfDay` from date-fns.
- `correlate(domainA: Domain, domainB: Domain, range: TimeRange, lagDays?: number): Promise<CorrelationResult>` -- fetches both domains, calls `correlateTimeSeries` from analytics-stats.

**Layer 2 -- Pre-built Queries:**
- `fluidBalance(range: TimeRange): Promise<AnalyticsResult<FluidBalanceResult>>` -- fetches water intake and urination records. Computes daily totals AND intraday time-series. Daily: each day gets intakeMl, urinationCount, urinationEstimatedMl, balance (intake - estimatedOutput), target (500ml above output per context). Intraday: every event as a DataPoint with running cumulative balance. Returns unit='ml'.
- `adherenceRate(range: TimeRange, prescriptionId?: string): Promise<AnalyticsResult<AdherenceResult>>` -- calls `getDoseScheduleForDateRange`, counts taken vs total slots. If prescriptionId provided, filters to that prescription. Returns daily breakdown + overall rate. Unit='ratio'.
- `bpTrend(range: TimeRange): Promise<AnalyticsResult<BPTrendResult>>` -- fetches BP records, computes separate trends for systolic and diastolic using `trend()` from analytics-stats. Returns readings + trend direction + averages. Unit='mmHg'.
- `weightTrend(range: TimeRange): Promise<AnalyticsResult<WeightTrendResult>>` -- fetches weight records, computes trend, min, max, avg. Unit='kg'.
- `saltVsWeight(range: TimeRange, lagDays?: number): Promise<AnalyticsResult<CorrelationResult>>` -- correlates salt intake with weight using configurable lag (default DEFAULT_SALT_WEIGHT_LAG_DAYS=2). Unit='correlation'.
- `caffeineVsBP(range: TimeRange): Promise<AnalyticsResult<CorrelationResult>>` -- correlates caffeine substance records with BP systolic. Unit='correlation'.
- `alcoholVsBP(range: TimeRange): Promise<AnalyticsResult<CorrelationResult>>` -- correlates alcohol substance records with BP systolic. Unit='correlation'. This is part of the locked full correlation kit.

**Critical rules:**
- NEVER import from `@/lib/db` -- only call domain services (intake-service, health-service, urination-service, dose-schedule-service, medication-service). Exception: for caffeine/alcohol domains, import from substance-service's getSubstanceRecordsByDateRange.
- All return types are plain objects, no class instances
- Use `format(date, 'yyyy-MM-dd')` from date-fns for day keys
- Use `startOfDay()` / `endOfDay()` from date-fns for midnight boundaries (NOT day-start-hour)
- Handle empty data gracefully: 0 records = neutral results (rate=0, empty arrays, 'stable' trends)

Create `src/lib/analytics-service.test.ts` with unit tests using fake-indexeddb:

- Test alcoholVsBP returns correlation result
- Test all results are JSON-serializable (JSON.parse(JSON.stringify(result)) deep-equals result)
- Test empty data returns neutral results (no errors)
- Mock domain services where needed to isolate analytics logic
