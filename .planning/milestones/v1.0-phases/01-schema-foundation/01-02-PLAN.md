# Plan 01-02: Dexie v10 Schema Rewrite

## Goal

`src/lib/db.ts` is rewritten to a single `db.version(10)` definition. All interfaces have `createdAt`, `updatedAt`, `deletedAt`, and `deviceId` fields. `currentStock` is removed from `InventoryItem` (deprecated as optional with `@deprecated` JSDoc). The v10 upgrade callback backfills timestamps on all existing records and converts legacy `currentStock` values into "initial" `inventoryTransactions`. All compound indexes required for cross-domain date-range queries are defined.

## Context

### Current State (from `src/lib/db.ts`)

The file defines versions 4 through 9 with upgrade callbacks. Key facts:
- Version 8 migrated from old `medications`/`medicationSchedules` tables → `prescriptions`/`medicationPhases`/`phaseSchedules`/`inventoryItems`. Those old tables are now empty.
- Version 9 added `inventoryTransactions` and decoupled dosage/strength in phases and schedules.
- `InventoryItem.currentStock: number` exists as a mutable counter — must become a deprecated optional field, NOT removed (TypeScript breaking change that blocks Phase 3).
- `InventoryTransaction.type` is `"refill" | "consumed" | "adjusted"` — needs `"initial"` added.
- `DoseLog` has no `createdAt`/`updatedAt`/`deletedAt`/`deviceId` — must be added.
- `Prescription` already has `createdAt` and `updatedAt` — needs `deletedAt` and `deviceId`.
- `AuditLog` has no sync-readiness fields — must be added.
- `medications` and `medicationSchedules` tables: empty after v8 upgrade. Omit from v10 stores — Dexie will delete them, which is correct.

### Locked Decisions (from CONTEXT.md)

- Single `db.version(10)` — delete all v4-v9 version blocks.
- ALL tables get `createdAt`, `updatedAt`, `deletedAt`, `deviceId`.
- `deletedAt` uses `null` (not `undefined`) so it is indexable.
- `currentStock` → deprecated optional (`currentStock?: number`) with `@deprecated` JSDoc. Services may still read it until Phase 3 fixes them; new code must not write to it.
- Every `InventoryItem` with `currentStock > 0` gets an "initial" `InventoryTransaction` created during upgrade.
- `deviceId` is used (NOT `realmId`) — more portable for the custom NeonDB sync layer.
- Index design: compound indexes on every table for date-range correlation queries.

### Critical Pitfalls to Avoid

1. **Must include `.upgrade()` callback** — even if upgrading from v9 (one version jump). Without it, Dexie silently deletes all object stores in the old schema when applying the new one, destroying user data.
2. **`deletedAt: null` vs `undefined`** — the upgrade callback must use `modify()` to set `deletedAt = null` on ALL existing records. `undefined` means absent; absent fields fail `WHERE deletedAt = null` queries.
3. **Repeat ALL store definitions** — if any table is omitted from `db.version(10).stores({...})`, Dexie deletes it.
4. **Omit `medications` and `medicationSchedules`** — they are empty after v8. Omitting from v10 deletes them (correct behavior). Including them would perpetuate dead schema.

<instructions>

**Step 1: Delete all version blocks.**

Remove the entire content from the first `db.version(4).stores({` through to the closing `});` of `db.version(9)` including its `.upgrade()` callback. The Dexie instance declaration above them stays.

**Step 2: Add single v10 definition.**

Replace the deleted version blocks with the following v10 definition. Every table that existed in v9 must appear in the v10 stores call. `medications` and `medicationSchedules` are intentionally omitted (they were emptied in v8 and should be deleted).

```typescript
db.version(10).stores({
  // Health records — compound indexes for date-range correlation queries
  intakeRecords:           "id, [type+timestamp], timestamp, source, updatedAt",
  weightRecords:           "id, timestamp, updatedAt",
  bloodPressureRecords:    "id, timestamp, position, arm, updatedAt",
  eatingRecords:           "id, timestamp, updatedAt",
  urinationRecords:        "id, timestamp, updatedAt",
  defecationRecords:       "id, timestamp, updatedAt",

  // Medication domain — compound indexes for cross-domain queries
  prescriptions:           "id, isActive, updatedAt",
  medicationPhases:        "id, prescriptionId, status, type, updatedAt",
  phaseSchedules:          "id, phaseId, time, enabled, updatedAt",
  inventoryItems:          "id, prescriptionId, isActive, updatedAt",
  inventoryTransactions:   "id, [inventoryItemId+timestamp], inventoryItemId, timestamp, type, updatedAt",
  doseLogs:                "id, [prescriptionId+scheduledDate], prescriptionId, phaseId, scheduleId, scheduledDate, scheduledTime, status, updatedAt",
  dailyNotes:              "id, date, prescriptionId, doseLogId, updatedAt",

  // Audit and system
  auditLogs:               "id, [action+timestamp], timestamp, action",
}).upgrade(async (trans) => {
  const now = Date.now();
  const deviceId = "migrated-v10";

  // Helper: backfill sync fields on all records in a table
  const backfill = async (tableName: string, timestampField = "timestamp") => {
    await trans.table(tableName).toCollection().modify((record: any) => {
      if (record.createdAt == null) {
        record.createdAt = record[timestampField] ?? record.createdAt ?? now;
      }
      if (record.updatedAt == null) {
        record.updatedAt = record.createdAt;
      }
      if (!("deletedAt" in record) || record.deletedAt === undefined) {
        record.deletedAt = null;
      }
      if (record.deviceId == null) {
        record.deviceId = deviceId;
      }
    });
  };

  // Backfill all health record tables
  await backfill("intakeRecords");
  await backfill("weightRecords");
  await backfill("bloodPressureRecords");
  await backfill("eatingRecords");
  await backfill("urinationRecords");
  await backfill("defecationRecords");

  // Backfill medication tables
  await backfill("prescriptions", "createdAt");
  await backfill("medicationPhases", "createdAt");
  await backfill("phaseSchedules", "createdAt");
  await backfill("inventoryItems", "createdAt");
  await backfill("inventoryTransactions");
  await backfill("doseLogs", "actionTimestamp");
  await backfill("dailyNotes", "createdAt");
  await backfill("auditLogs");

  // Event-source inventory: convert legacy currentStock to "initial" transactions
  const items = await trans.table("inventoryItems").toArray();
  for (const item of items) {
    if (item.currentStock != null && item.currentStock > 0) {
      await trans.table("inventoryTransactions").add({
        id: crypto.randomUUID(),
        inventoryItemId: item.id,
        timestamp: item.createdAt ?? now,
        amount: item.currentStock,
        type: "initial",
        note: "Migrated from v9 currentStock field",
        doseLogId: undefined,
        createdAt: item.createdAt ?? now,
        updatedAt: item.createdAt ?? now,
        deletedAt: null,
        deviceId: deviceId,
      });
    }
    // Do not delete currentStock from the record — it stays as a deprecated
    // optional field. Services read it until Phase 3 removes those reads.
    // Setting it to undefined here would cause TypeScript errors in Phase 3
    // to surface correctly (undefined !== absent field in IDB).
  }
});
```
