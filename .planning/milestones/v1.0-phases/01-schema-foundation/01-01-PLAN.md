# Plan 01-01: Test Infrastructure Setup

## Goal

Vitest is installed, configured, and able to run tests that import `db.ts` via the `@/*` alias. A shared test setup file and synthetic fixture generators exist so that migration tests in Plan 01-03 have a foundation to build on.

This plan does NOT write any migration tests — it only erects the scaffolding. Success is `pnpm test` producing a green run with at least one smoke test passing.

## Context

- Project uses pnpm. Package manager is enforced via `preinstall` hook.
- `tsconfig.json` has `@/*` → `src/*` path alias already configured.
- No `vitest.config.ts` exists today. No `test` script in `package.json` yet.
- Playwright e2e tests live in `e2e/` and run via `pnpm test:e2e` — must not conflict.
- `fake-indexeddb/auto` patches the global IDB API for Node.js — must be imported before any Dexie code in test files.
- The Dexie instance in `db.ts` is a module singleton. Tests must reset it via `db.delete()` + `db.open()` in `beforeEach` or each test will bleed state into the next.

<instructions>
Create `vitest.config.ts` at the project root (NOT inside `src/`, NOT inside `next.config`):

```typescript
import { defineConfig } from "vitest/config";
import tsconfigPaths from "vite-tsconfig-paths";

export default defineConfig({
  plugins: [tsconfigPaths()],
  test: {
    environment: "node",
    setupFiles: ["src/__tests__/setup.ts"],
    include: ["src/**/*.test.ts"],
    exclude: ["e2e/**", "node_modules/**"],
    globals: false,
  },
});
```

Key decisions:
- `environment: "node"` — fake-indexeddb patches global IDB in Node; browser environment not needed
- `setupFiles` points to the setup file created in task 01-01-C
- `include` targets only `src/**/*.test.ts` — Playwright `e2e/` specs are excluded explicitly
- `tsconfigPaths()` plugin resolves `@/*` imports to `src/*` as per `tsconfig.json`
- `globals: false` — tests must import `describe`, `it`, `expect` from `vitest` explicitly (no implicit globals)
</instructions>

File exists at `vitest.config.ts` in the project root with the exact configuration above.

**File 1: `src/__tests__/setup.ts`**

```typescript
import "fake-indexeddb/auto";
import { db } from "@/lib/db";
import { beforeEach, afterAll } from "vitest";

// Reset database between every test to prevent state bleed
beforeEach(async () => {
  await db.delete();
  await db.open();
});

afterAll(async () => {
  await db.close();
});
```

The `import "fake-indexeddb/auto"` MUST be the first line. It patches the global `indexedDB`, `IDBKeyRange`, etc. before Dexie is loaded. If it appears after `db` is imported, Dexie will not find IDB and will throw.

**Smoke test to verify infrastructure:**

Create `src/__tests__/smoke.test.ts`:

```typescript
import { describe, it, expect } from "vitest";
import { db } from "@/lib/db";

describe("test infrastructure", () => {
  it("can open and read an empty database", async () => {
    const records = await db.intakeRecords.toArray();
    expect(records).toEqual([]);
  });

  it("can insert and retrieve a record", async () => {
    await db.intakeRecords.add({
      id: "smoke-test-id",
      type: "water",
      amount: 250,
      timestamp: Date.now(),
    } as any);
    const record = await db.intakeRecords.get("smoke-test-id");
    expect(record).toBeDefined();
    expect(record?.amount).toBe(250);
  });
});
```
