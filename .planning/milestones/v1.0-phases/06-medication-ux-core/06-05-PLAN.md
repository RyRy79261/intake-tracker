  **Bug fix: Boolean indexing (CRITICAL)**
  In Dexie, `.where("isActive").equals(1)` does NOT match `true` (boolean). All service files store `isActive: true` but query with `.equals(1)`. This causes new prescriptions to not appear in the schedule.

  Fix ALL occurrences across these files by replacing `.where("field").equals(1).toArray()` with `.where("field").equals(1).toArray()` -> use `.toArray()` then `.filter()` OR use `.filter()` directly. The safest approach: replace indexed boolean queries with `.toArray()` followed by `.filter(item => item.isActive === true)` (or `.enabled === true`). Specifically:

  - `medication-service.ts` line 218: `db.prescriptions.where("isActive").equals(1)` -> `db.prescriptions.filter(p => p.isActive === true)`
  - `medication-service.ts` line 232: `.where({ prescriptionId, isActive: 1 })` -> `.where("prescriptionId").equals(prescriptionId).filter(i => i.isActive === true)`
  - `medication-service.ts` line 241: `db.inventoryItems.where("isActive").equals(1)` -> `db.inventoryItems.filter(i => i.isActive === true)`
  - `medication-schedule-service.ts` line 28: same pattern for prescriptions
  - `medication-schedule-service.ts` line 37: same pattern for inventoryItems
  - `medication-schedule-service.ts` line 43: `.where("enabled").equals(1)` -> `.filter(s => s.enabled === true)` for phaseSchedules
  - `dose-schedule-service.ts` lines 111, 130, 152: same pattern for `.equals(1)` on boolean fields
  - `medication-notification-service.ts` lines 67, 73, 133: same pattern

  **New service functions for transaction editing:**
  Add to `medication-service.ts`:
  - `updateInventoryTransaction(id: string, updates: { amount?: number; note?: string }): Promise<ServiceResult<void>>` - Updates a transaction's amount/note, then recalculates currentStock for its parent inventory item by summing all non-deleted transactions.
  - `deleteInventoryTransaction(id: string): Promise<ServiceResult<void>>` - Soft-deletes a transaction (sets deletedAt), then recalculates currentStock for its parent inventory item.

  Both functions must:
  1. Look up the transaction to get `inventoryItemId`
  2. Perform the update/soft-delete
  3. Recalculate `currentStock` on the parent InventoryItem by summing all transactions where `deletedAt === null`
  4. Wrap in `db.transaction('rw', [db.inventoryTransactions, db.inventoryItems], ...)`
  5. Create audit log entry

  Add corresponding hooks to `use-medication-queries.ts`:
  - `useUpdateInventoryTransaction()`
  - `useDeleteInventoryTransaction()`
