Create `src/app/api/ai/interaction-check/route.ts`:

1. Define Zod request schema as a discriminated union on `mode` field:
   - `mode: "conflict"` — has `newMedication: string` (the generic name being added)
   - `mode: "lookup"` — has `substance: string` (ad-hoc query)
   - Both modes include `activePrescriptions: z.array(z.object({ genericName: z.string(), drugClass: z.string().optional() })).min(1)`

2. Define Zod response schema:
   ```
   InteractionResultSchema = z.object({
     interactions: z.array(z.object({
       substance: z.string(),
       medication: z.string(),
       severity: z.enum(["AVOID", "CAUTION", "OK"]),
       description: z.string(),
     })),
     drugClass: z.string().optional(),
     summary: z.string().optional(),
   })
   ```

3. System prompt: Clinical pharmacist assistant. Given patient's current medications and a substance to check, identify drug interactions. Return JSON with interactions array, drugClass, summary. Severity: AVOID (dangerous), CAUTION (monitor), OK (no significant interaction). Be precise and evidence-based, err on CAUTION when uncertain.

4. Follow titration-warnings pattern exactly:
   - `withAuth` wrapper
   - Zod request validation with 400 on failure
   - Check `process.env.PERPLEXITY_API_KEY`, return 503 if missing
   - Rate limiting with separate `rateLimitMap` (5 requests per 60 seconds per user)
   - `sanitizeForAI` on all user inputs (substance/medication names)
   - Build user prompt: list all active prescriptions as "Currently taking: genericName (drugClass)" then "Check: substance/newMedication"
   - Perplexity call with `model: "sonar-reasoning-pro"`, `temperature: 0.2`, `max_tokens: 2000`
   - JSON extraction: `content.match(/\{[\s\S]*\}/)`
   - Zod response validation
   - Retry once on failure
   - Return 502 on double failure
   - Wrap in try/catch, return 500 on unexpected errors

5. Export only POST.

Create `src/hooks/use-interaction-check.ts`:

1. Export `InteractionResult` type matching the API response:
   ```typescript
   export interface InteractionItem {
     substance: string;
     medication: string;
     severity: "AVOID" | "CAUTION" | "OK";
     description: string;
   }
   export interface InteractionResult {
     interactions: InteractionItem[];
     drugClass?: string;
     summary?: string;
   }
   ```

2. Export `useInteractionCheck()` hook:
   - Returns `{ check, data, isLoading, error, reset }`
   - `check(params: { mode: "conflict"; newMedication: string; activePrescriptions: {...}[] } | { mode: "lookup"; substance: string; activePrescriptions: {...}[] })` triggers the API call
   - For `mode: "lookup"`, check `getCached(substance)` first. If hit, set data immediately without API call.
   - On API success for lookup mode, call `setCache(substance, result)`.
   - Uses `useState` for data/loading/error (not React Query — these are imperative one-shot calls, not reactive queries).
   - Uses `AbortController` with 15-second timeout. On timeout, set error to "Interaction check timed out" and return.
   - `reset()` clears data and error state.

3. Export `useRefreshInteractions()` hook:
   - Returns `{ refresh, isRefreshing }`
   - `refresh(prescriptionId: string, genericName: string, activePrescriptions: {...}[])` calls the API with `mode: "conflict"` and `newMedication: genericName`
   - On success, calls `useUpdatePrescription` to update the prescription's `contraindications` and `warnings` fields from the response:
     - `contraindications`: interactions with severity "AVOID" mapped to `"${interaction.medication}: ${interaction.description}"`
     - `warnings`: interactions with severity "CAUTION" mapped to same format
     - If `drugClass` returned, prepend `"Drug class: ${drugClass}"` to warnings array
   - Uses the `useUpdatePrescription` mutation from `use-medication-queries.ts`

4. All hooks are client-side only (`"use client"` not needed in hooks file — the consuming component handles that).
