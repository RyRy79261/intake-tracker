    1. Install dependencies: `pnpm add web-push @neondatabase/serverless`
    2. Install types: `pnpm add -D @types/web-push`

    3. Create `scripts/generate-vapid-keys.js`:
       - Use `require("web-push").generateVAPIDKeys()`
       - Print `NEXT_PUBLIC_VAPID_PUBLIC_KEY=...` and `VAPID_PRIVATE_KEY=...` to stdout
       - Add usage comment: "Run: node scripts/generate-vapid-keys.js"

    4. Create `scripts/push-migration.sql` with 4 tables:
       - `push_subscriptions` (id SERIAL PK, user_id TEXT NOT NULL UNIQUE, endpoint TEXT NOT NULL, p256dh TEXT NOT NULL, auth_key TEXT NOT NULL, created_at TIMESTAMPTZ DEFAULT NOW(), updated_at TIMESTAMPTZ DEFAULT NOW())
       - `push_dose_schedules` (id SERIAL PK, user_id TEXT NOT NULL, time_slot TEXT NOT NULL, day_of_week INTEGER NOT NULL, medications_json TEXT NOT NULL, UNIQUE(user_id, time_slot, day_of_week))
       - `push_sent_log` (id SERIAL PK, user_id TEXT NOT NULL, time_slot TEXT NOT NULL, sent_date DATE NOT NULL, follow_up_index INTEGER NOT NULL DEFAULT 0, sent_at TIMESTAMPTZ DEFAULT NOW(), UNIQUE(user_id, time_slot, sent_date, follow_up_index))
       - `push_settings` (user_id TEXT PK, enabled BOOLEAN NOT NULL DEFAULT true, follow_up_count INTEGER NOT NULL DEFAULT 2, follow_up_interval_minutes INTEGER NOT NULL DEFAULT 10, day_start_hour INTEGER NOT NULL DEFAULT 2)

    5. Create `src/lib/push-sender.ts`:
       - Import webpush from "web-push"
       - Call `webpush.setVapidDetails("mailto:notifications@intake-tracker.app", process.env.NEXT_PUBLIC_VAPID_PUBLIC_KEY!, process.env.VAPID_PRIVATE_KEY!)`
       - Export async function `sendPush(subscription: { endpoint: string; keys: { p256dh: string; auth: string } }, payload: string): Promise<{ success: boolean; statusCode?: number }>` that wraps `webpush.sendNotification` with TTL: 600, urgency: "high", catches errors, returns `{ success: false, statusCode: 410 }` on 410 Gone
       - Re-export webpush for direct use if needed

    6. Create `src/lib/push-db.ts`:
       - Import `neon` from `@neondatabase/serverless`
       - Helper: `function getSQL() { return neon(process.env.DATABASE_URL!); }`
       - `savePushSubscription(userId: string, sub: { endpoint: string; keys: { p256dh: string; auth: string } })` — UPSERT into push_subscriptions
       - `deletePushSubscription(userId: string)` — DELETE from push_subscriptions WHERE user_id
       - `getDueNotifications(currentTime: string, dayOfWeek: number, today: string)` — SELECT subscriptions JOIN dose_schedules WHERE time_slot matches AND NOT EXISTS in sent_log with follow_up_index=0
       - `getFollowUpNotifications(today: string, followUpIndex: number, intervalMinutes: number)` — SELECT from sent_log JOIN subscriptions WHERE sent_date=today AND follow_up_index = followUpIndex-1 AND sent_at <= NOW() - intervalMinutes AND NOT EXISTS sent_log with current followUpIndex
       - `logSentNotification(userId: string, timeSlot: string, sentDate: string, followUpIndex: number)` — INSERT into push_sent_log with ON CONFLICT DO NOTHING
       - `syncDoseSchedules(userId: string, schedules: Array<{ timeSlot: string; dayOfWeek: number; medicationsJson: string }>)` — DELETE all for user_id, then INSERT batch
       - `getSettings(userId: string)` — SELECT from push_settings, return defaults if not found
       - `saveSettings(userId: string, settings: { enabled: boolean; followUpCount: number; followUpIntervalMinutes: number; dayStartHour: number })` — UPSERT into push_settings
       - All functions use the neon SQL tagged template literal syntax: ``sql`SELECT ...` ``
