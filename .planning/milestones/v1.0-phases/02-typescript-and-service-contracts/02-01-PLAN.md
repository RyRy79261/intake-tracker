    1. Create `src/lib/service-result.ts` with the following:
       ```typescript
       export type ServiceResult<T> =
         | { success: true; data: T }
         | { success: false; error: string; details?: unknown };

       export function ok<T>(data: T): ServiceResult<T> {
         return { success: true, data };
       }

       export function err<T = never>(error: string, details?: unknown): ServiceResult<T> {
         return { success: false, error, details };
       }
       ```

    Fix all TypeScript errors in `src/lib/` files (~50 errors) AND refactor service functions to return `ServiceResult<T>`.

    **TypeScript error fix patterns (from RESEARCH.md):**
    - `noUncheckedIndexedAccess` errors: Array element access now returns `T | undefined`. Fix with:
      - `for...of` loops instead of indexed `for` loops where possible
      - Explicit null checks: `const item = arr[0]; if (!item) return err("not found");`
      - `.at(0)` which already types as `T | undefined`
    - `exactOptionalPropertyTypes` errors: Optional properties cannot receive explicit `undefined`. Fix with:
      - Conditional spread: `...(value !== undefined && { field: value })`
      - Add `| undefined` to the target type ONLY if undefined is a genuinely valid value
    - Do NOT use `// @ts-ignore`, `// @ts-expect-error`, or `as any` for ANY fix

    **ServiceResult refactoring pattern:**
    For each public service function that currently throws or returns raw data:
    1. Import `{ ok, err, ServiceResult }` from `@/lib/service-result`
    2. Wrap the function body in try/catch
    3. Return `ok(data)` on success
    4. Return `err(message, details)` on failure
    5. Keep internal `logAudit()` calls inside the service (not in hooks)
    6. Migrate one service file at a time, verify it compiles before moving to the next

    **`any` elimination in services:**
    - `src/lib/db.ts` has 1 `any` in a migration callback -- this is acceptable, leave it
    - Fix any other `any` in service files with proper types

    **Important:** Only fix errors in `src/lib/` files. Leave component/page errors for Plan 02-02. Run `npx tsc --noEmit 2>&1 | grep "src/lib/"` to verify only non-lib errors remain.

    **Service-to-service imports:** Keep the existing unidirectional pattern (dose-log-service -> medication-service, etc.). Do not create circular imports. Per RESEARCH.md discretionary recommendation: no ESLint rule for service-to-service -- the existing pattern is clean.
