**Replace useQuery read hooks with useLiveQuery:**
```typescript
// BEFORE:
import { useQuery, useMutation, useQueryClient } from "@tanstack/react-query";

  return useQuery({ queryKey: [...], queryFn: async () => unwrap(await getData()) });
}

import { useLiveQuery } from "dexie-react-hooks";
import { useMutation } from "@tanstack/react-query";

  return useLiveQuery(() => getData(), []); // getData now returns T directly
}
```

**For array returns, provide default value:**
```typescript
export function useRecords() {
  return useLiveQuery(() => getRecords(), [], []); // default to empty array
}
```

**Remove all invalidation from mutation hooks:**
- Delete `useInvalidate*()` helper functions
- Delete `onSuccess: invalidate` from all useMutation calls
- Delete `queryClient.invalidateQueries(...)` calls
- Remove `useQueryClient` imports if no longer needed
- Remove query key factories (intakeKeys, graphKeys, etc.) if no longer used

**Keep useMutation for writes:**
- Mutations still use `useMutation({ mutationFn: async (...) => unwrap(await mutate(...)) })`
- No onSuccess invalidation needed — useLiveQuery auto-detects DB changes

**Special: use-intake-queries.ts rolling 24h timer:**
- The current pattern uses setInterval to invalidateQueries every 60 seconds
- New pattern: use `useState` with a `currentMinute` counter that updates every 60 seconds. Pass it as a dep to useLiveQuery so the query re-runs on the minute boundary:
```typescript
export function useIntakeTotal(type: "water" | "salt") {
  const [tick, setTick] = useState(0);
  useEffect(() => {
    const interval = setInterval(() => setTick(t => t + 1), 60_000);
    return () => clearInterval(interval);
  }, []);
  return useLiveQuery(() => getTotalInLast24Hours(type), [type, tick], 0);
}
```
- Apply same pattern to `useDailyIntakeTotal`, `useIntakeRecords`.
- The `useIntake` combined hook: simplify to use the useLiveQuery-based hooks. Remove `refresh` callback (no longer needed). Keep `addRecord` and `removeRecord` via mutations. Return shape should remain backward-compatible: `{ dailyTotal, rollingTotal, total, isLoading, addRecord, removeRecord }`. For `isLoading`, derive from `dailyTotalResult === undefined`.

**Special: use-graph-data.ts:**

```typescript
export function useGraphData(scope: GraphScope) {

```
- Keep `fetchGraphData` function as-is (it calls service read functions that now return T directly — update the calls to remove `unwrap()` since services return T directly now).
- Export `graphKeys` can be removed (no longer used for invalidation). But check if anything imports it — if so, keep as deprecated empty object or remove the import from consumers.
