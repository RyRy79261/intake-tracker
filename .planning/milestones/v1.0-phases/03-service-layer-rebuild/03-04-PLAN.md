Create `src/lib/dose-schedule-service.ts` — the derive-at-read daily dose schedule service (Claude's discretion choice: derive from schedule at read time, not pre-create pending records).

**Export types:**
```typescript
export type DoseSlotStatus = "taken" | "skipped" | "pending" | "missed";

export interface DoseSlot {
  // Schedule info
  prescriptionId: string;
  phaseId: string;
  scheduleId: string;
  scheduledDate: string;           // YYYY-MM-DD
  scheduleTimeUTC: number;         // minutes from midnight UTC
  localTime: string;               // "HH:MM" in device timezone for display
  dosageMg: number;                // from PhaseSchedule.dosage
  unit: string;                    // from MedicationPhase.unit

  // Status
  status: DoseSlotStatus;
  existingLog?: DoseLog;           // the actual log record if one exists

  // Related entities for display
  prescription: Prescription;
  phase: MedicationPhase;
  schedule: PhaseSchedule;
  inventory?: InventoryItem;       // active inventory for this prescription

  // Computed stock info
  pillsPerDose?: number;           // dosageMg / inventory.strength (if inventory exists)
  inventoryWarning?: string;       // "negative_stock" | "no_inventory" | "odd_fraction"
}
```

**Main function:**
```typescript
export async function getDailyDoseSchedule(
  dateStr: string,
  timezone?: string  // defaults to getDeviceTimezone()
): Promise<DoseSlot[]>
```

Implementation:
1. Parse dateStr to get day-of-week (use `new Date(dateStr + "T12:00:00")` to avoid timezone shift issues with date parsing)
2. Get all active prescriptions, their active phases, and phase schedules for this day-of-week (similar to existing getDailySchedule logic but integrated)
3. Get all existing dose logs for this date: `db.doseLogs.where("scheduledDate").equals(dateStr).toArray()`
4. Get all active inventory items
5. For each schedule entry that applies to this date:
   - Look up matching dose log by prescriptionId + phaseId + scheduleId + date
   - Determine status:
     - If dose log exists: use its status ("taken", "skipped", "rescheduled")
     - If no log and date is today: "pending"
     - If no log and date is in the past: "missed"
   - Convert scheduleTimeUTC to local display time using `formatLocalTime(schedule.scheduleTimeUTC, tz)`
   - Calculate pillsPerDose if inventory exists: `dosageMg / inventory.strength` (rounded to 4 decimals)
   - Set inventoryWarning if: no active inventory → "no_inventory", stock would go negative → "negative_stock", odd fraction → "odd_fraction"
6. Sort by localTime ascending
7. Return DoseSlot array

**Helper function:**
```typescript
export async function getDoseScheduleForDateRange(
  startDate: string,
  endDate: string,
  timezone?: string
): Promise<Map<string, DoseSlot[]>>
```
Returns a map of date → DoseSlot[] for a range (useful for history/calendar views). Iterate dates in range, call getDailyDoseSchedule for each.

**Note:** This function reads from 5 tables (prescriptions, medicationPhases, phaseSchedules, doseLogs, inventoryItems). When wrapped in useLiveQuery, it will re-run whenever any of these tables change. This is acceptable — IndexedDB reads are fast (<10ms per research).

export function usePrescriptions() {
  return useLiveQuery(() => getPrescriptions(), [], []);

  return useLiveQuery(() => getDailyDoseSchedule(dateStr), [dateStr]);
