# Verification — 01-liquids-water-input

**Verdict:** minor-gaps  ·  checked 118 claims, verified 110.

Document covers the Liquids/Water card unit. I read every file under "Files covered:" in full (liquids-card, water-tab, beverage-tab, preset-tab, manual-input-dialog, collapsible-time-input, recent-entries-list, use-intake-queries, use-substance-queries, use-composable-entry, substance-service, composable-entry-service, intake-service, alcohol-units, progress-utils, card-themes, constants, settings-store, utils, db, date-utils) plus related code it pulls in (use-edit-record, use-delete-with-toast, use-undo-delete-mutation, undo-toast, auth-guard, optional-trackers, use-now-tick, the AI substance-lookup route + schema, sync-queue). The document is largely accurate and unusually precise on numeric values (presets, constants, thresholds all check digit-for-digit). The defects are a few overstated edge-case claims (edit-form time cap, single-vs-two-decimal rounding, the delete-toast text) and a couple of minor mismatches.
